# Semantic HTML5 Elements

Before HTML5, developers built web pages using `<div>` and `<span>` tags with classes like `<div class="header">` or `<div id="navigation">`. The code worked, but it had no meaning. HTML5 changed this by introducing **semantic elements** - tags that describe what they contain.

Semantic HTML makes your code easier to read, improves accessibility for users with disabilities, boosts SEO rankings, and creates a better structure for your pages. In this lesson, you'll learn the core semantic HTML5 elements and how to use them correctly.

## What Is Semantic HTML?

**Semantic HTML** uses HTML elements that clearly describe their content's meaning and purpose, both to developers and to browsers.

Compare these two examples:

**Non-Semantic (Bad):**
```html
<div class="header">
  <div class="nav">
    <a href="/">Home</a>
    <a href="/about">About</a>
  </div>
</div>
<div class="content">
  <div class="post">
    <h1>My Blog Post</h1>
    <p>Content here...</p>
  </div>
</div>
```

**Semantic (Good):**
```html
<header>
  <nav>
    <a href="/">Home</a>
    <a href="/about">About</a>
  </nav>
</header>
<main>
  <article>
    <h1>My Blog Post</h1>
    <p>Content here...</p>
  </article>
</main>
```

The semantic version tells you exactly what each part does, without reading CSS classes.

## Core Semantic HTML5 Elements

Let's explore the essential semantic elements you'll use in every project.

### `<header>` - Introductory Content

The `<header>` element represents introductory content or navigation. It typically contains:
- Site logo
- Main navigation
- Search bar
- Tagline or description

**Example - Site Header:**
```html
<header>
  <img src="logo.png" alt="Company Logo">
  <nav>
    <a href="/">Home</a>
    <a href="/products">Products</a>
    <a href="/contact">Contact</a>
  </nav>
</header>
```

**Example - Article Header:**
```html
<article>
  <header>
    <h1>Understanding JavaScript Closures</h1>
    <p>Published on <time datetime="2026-01-14">January 14, 2026</time></p>
    <p>By Sarah Chen</p>
  </header>
  <p>Article content...</p>
</article>
```

**Important:** You can have multiple `<header>` elements on a page - one for the site, and others for individual `<article>` or `<section>` elements.

### `<nav>` - Navigation Links

The `<nav>` element represents a section of navigation links. Use it for major navigation blocks.

**Example - Main Navigation:**
```html
<nav>
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/about">About Us</a></li>
    <li><a href="/services">Services</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>
```

**Example - Footer Navigation:**
```html
<footer>
  <nav aria-label="Footer Navigation">
    <a href="/privacy">Privacy Policy</a>
    <a href="/terms">Terms of Service</a>
    <a href="/sitemap">Sitemap</a>
  </nav>
</footer>
```

**When to Use:**
- Main site navigation
- Table of contents
- Pagination controls
- Footer links

**When NOT to Use:**
- Every group of links (like social media icons)
- Links within paragraphs

### `<main>` - Primary Content

The `<main>` element contains the primary content of the page. Each page should have only **one** `<main>` element, and it should not be inside `<article>`, `<aside>`, `<footer>`, `<header>`, or `<nav>`.

**Example:**
```html
<body>
  <header>
    <!-- Site header with logo and navigation -->
  </header>

  <main>
    <!-- Primary page content goes here -->
    <h1>Welcome to Our Site</h1>
    <p>This is the main content...</p>
  </main>

  <footer>
    <!-- Footer content -->
  </footer>
</body>
```

**Important Rule:** Only one `<main>` per page, and skip navigation and repeated content.

### `<article>` - Self-Contained Content

The `<article>` element represents a self-contained piece of content that could stand alone. Ask yourself: "Could I syndicate this content (RSS feed, repost elsewhere)?" If yes, use `<article>`.

**Good Uses:**
- Blog posts
- News articles
- Forum posts
- Product cards
- User comments

**Example - Blog Post:**
```html
<article>
  <header>
    <h2>10 Tips for Better CSS</h2>
    <p>By John Doe | January 10, 2026</p>
  </header>

  <p>CSS can be tricky, but these tips will help...</p>

  <section>
    <h3>1. Use CSS Variables</h3>
    <p>CSS variables make theming easy...</p>
  </section>

  <footer>
    <p>Tags: CSS, Web Development</p>
  </footer>
</article>
```

**Example - Product Card:**
```html
<article>
  <img src="product.jpg" alt="Product Name">
  <h3>Wireless Headphones</h3>
  <p>$99.99</p>
  <button>Add to Cart</button>
</article>
```

### `<section>` - Thematic Grouping

The `<section>` element groups related content by theme. Unlike `<article>`, a section is NOT self-contained - it's part of a larger whole.

**Example - About Page Sections:**
```html
<main>
  <h1>About Our Company</h1>

  <section>
    <h2>Our Mission</h2>
    <p>We strive to...</p>
  </section>

  <section>
    <h2>Our Team</h2>
    <p>Meet the people behind...</p>
  </section>

  <section>
    <h2>Our History</h2>
    <p>Founded in 2010...</p>
  </section>
</main>
```

**Rule of Thumb:**
- Use `<article>` if the content could stand alone
- Use `<section>` if it's part of a larger document
- Use `<div>` if you just need styling with no semantic meaning

### `<aside>` - Tangential Content

The `<aside>` element contains content that's tangentially related to the main content - like a sidebar. It's related, but not essential.

**Example - Sidebar:**
```html
<main>
  <article>
    <h1>JavaScript Basics</h1>
    <p>JavaScript is a programming language...</p>
  </article>

  <aside>
    <h3>Related Articles</h3>
    <ul>
      <li><a href="/html-basics">HTML Basics</a></li>
      <li><a href="/css-basics">CSS Basics</a></li>
    </ul>
  </aside>
</main>
```

**Example - Pull Quote:**
```html
<article>
  <p>The web is constantly evolving...</p>

  <aside>
    <blockquote>
      "The best way to predict the future is to invent it."
      - Alan Kay
    </blockquote>
  </aside>

  <p>As web developers, we must...</p>
</article>
```

**Good Uses:**
- Sidebars
- Pull quotes
- Related posts
- Author bio
- Advertisements

### `<footer>` - Footer Content

The `<footer>` element represents footer content for its parent element. Like `<header>`, you can have multiple footers on a page.

**Example - Site Footer:**
```html
<footer>
  <p>&copy; 2026 My Website. All rights reserved.</p>
  <nav>
    <a href="/privacy">Privacy Policy</a>
    <a href="/terms">Terms</a>
    <a href="/contact">Contact</a>
  </nav>
</footer>
```

**Example - Article Footer:**
```html
<article>
  <h2>How to Learn JavaScript</h2>
  <p>Article content...</p>

  <footer>
    <p>Author: Jane Smith</p>
    <p>Published: January 14, 2026</p>
    <p>Tags: JavaScript, Tutorial, Web Development</p>
  </footer>
</article>
```

## Complete Page Structure Example

Here's how these elements work together:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Blog</title>
</head>
<body>
  <!-- Site Header -->
  <header>
    <h1>My Awesome Blog</h1>
    <nav>
      <a href="/">Home</a>
      <a href="/about">About</a>
      <a href="/contact">Contact</a>
    </nav>
  </header>

  <!-- Main Content -->
  <main>
    <!-- Blog Post -->
    <article>
      <header>
        <h2>Understanding HTML5 Semantic Elements</h2>
        <p>Published on <time datetime="2026-01-14">January 14, 2026</time></p>
      </header>

      <section>
        <h3>What Are Semantic Elements?</h3>
        <p>Semantic elements clearly describe their meaning...</p>
      </section>

      <section>
        <h3>Why Use Them?</h3>
        <p>They improve accessibility and SEO...</p>
      </section>

      <footer>
        <p>Author: John Developer</p>
        <p>Tags: HTML5, Web Development, Accessibility</p>
      </footer>
    </article>

    <!-- Sidebar -->
    <aside>
      <h3>About the Author</h3>
      <p>John is a web developer with 10 years of experience...</p>

      <h3>Related Posts</h3>
      <ul>
        <li><a href="/css-basics">CSS Basics</a></li>
        <li><a href="/javascript-intro">JavaScript Introduction</a></li>
      </ul>
    </aside>
  </main>

  <!-- Site Footer -->
  <footer>
    <p>&copy; 2026 My Blog. All rights reserved.</p>
    <nav aria-label="Footer Navigation">
      <a href="/privacy">Privacy Policy</a>
      <a href="/terms">Terms of Service</a>
    </nav>
  </footer>
</body>
</html>
```

## Best Practices

1. **Use semantic elements first, `<div>` only for styling**
   - If there's a semantic element that fits, use it
   - Use `<div>` and `<span>` only when no semantic element applies

2. **One `<main>` per page**
   - Each page should have exactly one `<main>` element
   - It should contain the primary content, not repeated navigation or footers

3. **Multiple `<header>` and `<footer>` are OK**
   - You can have a site header and article headers
   - Each `<header>` serves its parent element

4. **Nest elements properly**
   - `<main>` should not be inside `<article>`, `<aside>`, `<footer>`, `<header>`, or `<nav>`
   - `<article>` can contain `<section>`, `<header>`, and `<footer>`

5. **Add ARIA labels when you have multiple navigation blocks**
   - Use `aria-label` to distinguish navigation sections

## Common Mistakes to Avoid

1. **Using `<section>` or `<article>` without a heading**
   - Every `<section>` and `<article>` should have a heading (`<h1>` - `<h6>`)
   - This helps screen readers and SEO

2. **Wrapping everything in `<div>` out of habit**
   - Check if there's a semantic element first
   - `<div>` should be your last choice, not your first

3. **Multiple `<main>` elements**
   - Only one `<main>` per page
   - If you have multiple content areas, use `<section>` or `<article>` inside `<main>`

4. **Using `<article>` for non-standalone content**
   - Ask: "Could this be syndicated or reposted elsewhere?"
   - If no, use `<section>` instead

## Key Takeaways

- **Semantic HTML** uses elements that describe their content's meaning
- **`<header>`** contains introductory content (logo, navigation, title)
- **`<nav>`** wraps major navigation blocks
- **`<main>`** contains the primary page content (only one per page)
- **`<article>`** represents self-contained, standalone content
- **`<section>`** groups thematically related content
- **`<aside>`** contains tangentially related content (sidebars)
- **`<footer>`** contains footer information
- Semantic HTML improves **accessibility**, **SEO**, and **maintainability**
- Use semantic elements first; use `<div>` only when no semantic element fits

Semantic HTML is the foundation of modern, accessible web development. Master these elements, and you'll write cleaner, more professional code that works better for everyone.
