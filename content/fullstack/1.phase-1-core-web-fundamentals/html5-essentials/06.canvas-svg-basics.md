# Canvas & SVG Basics

HTML5 provides two powerful ways to create graphics on the web: **Canvas** and **SVG**. Both can create shapes, animations, and interactive graphics, but they work very differently. Choosing the right one depends on your use case.

In this lesson, you'll learn the fundamentals of both technologies, understand their strengths and weaknesses, and know when to use each one.

### Key Differences

| Feature | Canvas | SVG |
|---------|--------|-----|
| **Type** | Raster (bitmap) | Vector |
| **How it works** | Draws pixels | Creates shape objects |
| **Scaling** | Becomes pixelated | Stays sharp at any size |
| **DOM** | Not in DOM | Each element in DOM |
| **Events** | Manual hit detection | Built-in event listeners |
| **Performance** | Better for many objects | Slower with many objects |
| **Modification** | Redraw everything | Change individual elements |
| **Best for** | Games, photo editing, charts | Logos, icons, interactive diagrams |

## HTML5 Canvas API

**Canvas** is like a digital painting surface. You use JavaScript to draw shapes, text, and images pixel by pixel.

### Creating a Canvas

```html
<canvas id="myCanvas" width="500" height="300"></canvas>
```

**Important:** Set width and height in HTML attributes, not CSS. CSS just stretches the canvas (causing pixelation).

### Getting the Drawing Context

Before you can draw, you need the 2D context:

```javascript
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

// Now you can draw
ctx.fillStyle = 'blue';
ctx.fillRect(10, 10, 100, 50); // x, y, width, height
```

### Drawing Shapes

#### Rectangles

```javascript
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

// Filled rectangle
ctx.fillStyle = 'blue';
ctx.fillRect(50, 50, 150, 100); // x, y, width, height

// Outlined rectangle
ctx.strokeStyle = 'red';
ctx.lineWidth = 3;
ctx.strokeRect(250, 50, 150, 100);

// Clear rectangle (erase)
ctx.clearRect(75, 75, 100, 50);
```

#### Circles and Arcs

```javascript
// Full circle
ctx.beginPath();
ctx.arc(250, 150, 50, 0, Math.PI * 2); // x, y, radius, startAngle, endAngle
ctx.fillStyle = 'green';
ctx.fill();

// Half circle (arc)
ctx.beginPath();
ctx.arc(400, 150, 50, 0, Math.PI); // 0 to œÄ = half circle
ctx.strokeStyle = 'purple';
ctx.lineWidth = 3;
ctx.stroke();
```

#### Lines and Paths

```javascript
// Draw a triangle
ctx.beginPath();
ctx.moveTo(100, 200);    // Starting point
ctx.lineTo(150, 250);    // Line to second point
ctx.lineTo(50, 250);     // Line to third point
ctx.closePath();         // Close back to starting point
ctx.fillStyle = 'orange';
ctx.fill();
```

### Drawing Text

```javascript
// Filled text
ctx.font = '30px Arial';
ctx.fillStyle = 'black';
ctx.fillText('Hello Canvas!', 50, 50);

// Outlined text
ctx.strokeStyle = 'blue';
ctx.lineWidth = 2;
ctx.strokeText('Outlined Text', 50, 100);

// Text alignment
ctx.textAlign = 'center';
ctx.fillText('Centered Text', canvas.width / 2, 150);
```

### Drawing Images

```javascript
const img = new Image();
img.onload = function() {
  // Draw image at x, y
  ctx.drawImage(img, 50, 50);

  // Draw image with custom size
  ctx.drawImage(img, 50, 200, 200, 100); // x, y, width, height

  // Crop and draw image
  // drawImage(img, sx, sy, sw, sh, dx, dy, dw, dh)
  ctx.drawImage(img, 0, 0, 100, 100, 300, 50, 150, 150);
};
img.src = 'image.jpg';
```

### Canvas Animation

Animate by clearing and redrawing the canvas repeatedly:

```javascript
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

let x = 0;
let y = 150;
let dx = 2; // Speed

function animate() {
  // Clear canvas
  ctx.clearRect(0, 0, canvas.width, canvas.height);

  // Draw circle
  ctx.beginPath();
  ctx.arc(x, y, 20, 0, Math.PI * 2);
  ctx.fillStyle = 'blue';
  ctx.fill();

  // Update position
  x += dx;

  // Bounce off edges
  if (x + 20 > canvas.width || x - 20 < 0) {
    dx = -dx;
  }

  // Continue animation
  requestAnimationFrame(animate);
}

animate();
```

## SVG (Scalable Vector Graphics)

**SVG** creates graphics using XML markup. Each shape is an element in the DOM, making it easy to style with CSS and manipulate with JavaScript.

### Inline SVG

Embed SVG directly in HTML:

```html
<svg width="400" height="300">
  <!-- Rectangle -->
  <rect x="50" y="50" width="150" height="100" fill="blue" />

  <!-- Circle -->
  <circle cx="300" cy="100" r="50" fill="green" />

  <!-- Line -->
  <line x1="50" y1="200" x2="350" y2="200" stroke="red" stroke-width="3" />

  <!-- Text -->
  <text x="200" y="250" font-size="24" fill="black" text-anchor="middle">
    Hello SVG!
  </text>
</svg>
```

### SVG Shapes

#### Rectangle

```html
<svg width="200" height="100">
  <!-- Basic rectangle -->
  <rect x="10" y="10" width="180" height="80" fill="steelblue" />

  <!-- Rounded corners -->
  <rect x="10" y="10" width="180" height="80" rx="10" ry="10" fill="coral" />

  <!-- Outlined -->
  <rect x="10" y="10" width="180" height="80" fill="none" stroke="navy" stroke-width="3" />
</svg>
```

#### Circle and Ellipse

```html
<svg width="400" height="200">
  <!-- Circle -->
  <circle cx="100" cy="100" r="80" fill="gold" />

  <!-- Ellipse -->
  <ellipse cx="300" cy="100" rx="100" ry="60" fill="lightblue" />
</svg>
```

#### Line and Polyline

```html
<svg width="400" height="200">
  <!-- Single line -->
  <line x1="50" y1="50" x2="350" y2="150" stroke="red" stroke-width="3" />

  <!-- Multiple connected lines -->
  <polyline
    points="50,150 100,50 150,150 200,50 250,150"
    fill="none"
    stroke="blue"
    stroke-width="2"
  />
</svg>
```

#### Polygon

```html
<svg width="300" height="300">
  <!-- Triangle -->
  <polygon
    points="150,50 250,250 50,250"
    fill="orange"
    stroke="darkred"
    stroke-width="2"
  />

  <!-- Star -->
  <polygon
    points="150,20 180,100 260,100 200,150 220,230 150,180 80,230 100,150 40,100 120,100"
    fill="gold"
    stroke="orange"
    stroke-width="2"
  />
</svg>
```

#### Path (Advanced)

The `<path>` element creates complex shapes:

```html
<svg width="300" height="300">
  <!-- Curved path -->
  <path
    d="M 50 150 Q 150 50 250 150"
    fill="none"
    stroke="purple"
    stroke-width="3"
  />

  <!-- Heart shape -->
  <path
    d="M 150 200 L 100 150 Q 90 130 110 120 Q 130 110 150 130 Q 170 110 190 120 Q 210 130 200 150 Z"
    fill="red"
    stroke="darkred"
    stroke-width="2"
  />
</svg>
```

**Path Commands:**
- `M` - Move to
- `L` - Line to
- `Q` - Quadratic curve
- `C` - Cubic curve
- `Z` - Close path

### SVG Text

```html
<svg width="400" height="200">
  <!-- Basic text -->
  <text x="200" y="50" font-size="24" fill="navy" text-anchor="middle">
    SVG Text
  </text>

  <!-- Text along a path -->
  <defs>
    <path id="curve" d="M 50 150 Q 200 50 350 150" />
  </defs>
  <text font-size="18" fill="blue">
    <textPath href="#curve">
      This text follows a curve!
    </textPath>
  </text>
</svg>
```

### SVG with CSS

Style SVG elements with CSS:

```html
<style>
  .my-circle {
    fill: steelblue;
    stroke: navy;
    stroke-width: 3;
    transition: fill 0.3s;
  }

  .my-circle:hover {
    fill: coral;
    cursor: pointer;
  }

  .animated-rect {
    animation: pulse 2s infinite;
  }

  @keyframes pulse {
    0%, 100% { transform: scale(1); }
    50% { transform: scale(1.1); }
  }
</style>

<svg width="400" height="200">
  <circle class="my-circle" cx="100" cy="100" r="50" />
  <rect class="animated-rect" x="200" y="50" width="100" height="100" fill="purple" />
</svg>
```

### SVG with JavaScript

Manipulate SVG elements like HTML elements:

```html
<svg width="400" height="300">
  <circle id="myCircle" cx="200" cy="150" r="50" fill="blue" />
</svg>

<button onclick="changeColor()">Change Color</button>
<button onclick="moveCircle()">Move Circle</button>

<script>
  const circle = document.getElementById('myCircle');

  function changeColor() {
    const colors = ['red', 'blue', 'green', 'orange', 'purple'];
    const randomColor = colors[Math.floor(Math.random() * colors.length)];
    circle.setAttribute('fill', randomColor);
  }

  function moveCircle() {
    const x = Math.random() * 350 + 25; // 25 to 375
    const y = Math.random() * 250 + 25; // 25 to 275
    circle.setAttribute('cx', x);
    circle.setAttribute('cy', y);
  }

  // Event listener
  circle.addEventListener('click', () => {
    alert('Circle clicked!');
  });
</script>
```

## When to Use Canvas vs. SVG

### Use Canvas When:
- **Many objects:** Games with hundreds of sprites, particle effects
- **Pixel manipulation:** Photo editing, filters, effects
- **High-performance animations:** Frame-by-frame animations
- **Complex calculations:** Data visualizations with thousands of points
- **No need for interactivity:** Static renderings
- **Raster graphics:** Drawing photos or bitmap images

### Use SVG When:
- **Scalability matters:** Logos, icons that need to scale infinitely
- **Individual element interaction:** Click handlers, hover effects on shapes
- **Accessibility:** Screen readers can read SVG text
- **CSS styling:** You want to style shapes with CSS
- **SEO:** SVG text is indexable by search engines
- **Simple graphics:** Few shapes, not performance-critical
- **Responsive design:** Graphics that adapt to screen size

### Performance Comparison

| Objects | Canvas | SVG |
|---------|--------|-----|
| 1-100 | ‚ö°‚ö°‚ö° Fast | ‚ö°‚ö°‚ö° Fast |
| 100-1000 | ‚ö°‚ö°‚ö° Fast | ‚ö°‚ö° OK |
| 1000+ | ‚ö°‚ö°‚ö° Fast | ‚ö° Slow |
| 10,000+ | ‚ö°‚ö° OK | ‚ùå Too slow |

**Rule of Thumb:** Canvas for performance, SVG for scalability and interactivity.

## Accessibility Considerations

### Canvas Accessibility

Canvas is a bitmap, so it's invisible to screen readers. Always provide alternative content:

```html
<canvas id="chart" width="600" height="400" role="img" aria-label="Sales chart showing 25% growth">
  <!-- Fallback content for screen readers and old browsers -->
  <p>Sales increased by 25% in Q4, from $4M to $5M.</p>
</canvas>
```

### SVG Accessibility

SVG is more accessible because elements are in the DOM:

```html
<svg role="img" aria-labelledby="title desc">
  <title id="title">Sales Chart</title>
  <desc id="desc">Bar chart showing sales growth of 25% from Q3 to Q4.</desc>

  <!-- Chart elements... -->
</svg>
```

## Key Takeaways

- **Canvas** is raster-based (pixels); **SVG** is vector-based (shapes)
- **Canvas** is better for performance with many objects; **SVG** scales perfectly at any size
- **Canvas** requires redrawing everything; **SVG** elements can be individually modified
- **SVG elements** are in the DOM and can have event listeners; **Canvas** is just pixels
- Use **Canvas** for games, photo editing, and complex animations
- Use **SVG** for logos, icons, interactive diagrams, and responsive graphics
- **Canvas** draws with JavaScript API (`ctx.fillRect()`); **SVG** uses XML markup (`<rect>`)
- Both can be styled with CSS, but SVG is easier
- Always provide alternative text for accessibility (especially for Canvas)
- Performance: Canvas wins with 1000+ objects; SVG wins for simplicity and scalability

Canvas and SVG are both powerful graphics technologies. Choose based on your specific needs - performance vs. scalability, pixel manipulation vs. shape interaction. Often, the best approach is using both together in different parts of your application.
