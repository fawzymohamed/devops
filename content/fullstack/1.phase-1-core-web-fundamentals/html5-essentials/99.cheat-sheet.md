# HTML5 Essentials Cheat Sheet

Quick reference for HTML5 semantic elements, forms, accessibility, SEO, Web APIs, and graphics.

---

## Semantic HTML5 Elements

| Element | Purpose | When to Use |
|---------|---------|-------------|
| `<header>` | Introductory content | Page/section headers, navigation |
| `<nav>` | Navigation links | Main menus, table of contents |
| `<main>` | Primary content | One per page, unique content |
| `<article>` | Self-contained content | Blog posts, news articles |
| `<section>` | Thematic grouping | Chapters, tabbed content |
| `<aside>` | Related content | Sidebars, pull quotes |
| `<footer>` | Closing content | Copyright, contact info |
| `<figure>` | Self-contained media | Images, diagrams with captions |
| `<figcaption>` | Caption for figure | Image descriptions |

**Page Structure Template:**

```html
<body>
  <header>
    <nav><!-- Main navigation --></nav>
  </header>
  <main>
    <article>
      <section><!-- Content sections --></section>
    </article>
    <aside><!-- Sidebar --></aside>
  </main>
  <footer><!-- Footer content --></footer>
</body>
```

## Forms & Validation

### Input Types

| Type | Purpose | Example |
|------|---------|---------|
| `email` | Email validation | `<input type="email">` |
| `tel` | Phone numbers | `<input type="tel">` |
| `url` | URL validation | `<input type="url">` |
| `number` | Numeric input | `<input type="number" min="0" max="100">` |
| `range` | Slider control | `<input type="range" min="0" max="100">` |
| `date` | Date picker | `<input type="date">` |
| `time` | Time picker | `<input type="time">` |
| `color` | Color picker | `<input type="color">` |
| `search` | Search field | `<input type="search">` |

### Validation Attributes

| Attribute | Purpose | Example |
|-----------|---------|---------|
| `required` | Field must be filled | `<input required>` |
| `pattern` | Regex validation | `<input pattern="[A-Za-z]{3,}">` |
| `minlength` | Minimum characters | `<input minlength="8">` |
| `maxlength` | Maximum characters | `<input maxlength="100">` |
| `min` / `max` | Numeric range | `<input type="number" min="1" max="10">` |
| `step` | Increment value | `<input type="number" step="0.01">` |

### Form Best Practices

```html
<form novalidate>
  <label for="email">Email</label>
  <input type="email" id="email" name="email" required
         aria-describedby="email-hint">
  <span id="email-hint">We'll never share your email</span>
</form>
```

## Accessibility (A11y)

### ARIA Landmarks

| Role | Purpose | HTML5 Equivalent |
|------|---------|------------------|
| `banner` | Page header | `<header>` |
| `navigation` | Navigation | `<nav>` |
| `main` | Main content | `<main>` |
| `complementary` | Sidebar | `<aside>` |
| `contentinfo` | Footer | `<footer>` |

### Essential ARIA Attributes

| Attribute | Purpose | Example |
|-----------|---------|---------|
| `aria-label` | Accessible name | `<button aria-label="Close">X</button>` |
| `aria-labelledby` | Reference to label | `<div aria-labelledby="title-id">` |
| `aria-describedby` | Reference to description | `<input aria-describedby="hint">` |
| `aria-hidden` | Hide from AT | `<span aria-hidden="true">icon</span>` |
| `aria-expanded` | Expandable state | `<button aria-expanded="false">` |
| `aria-live` | Dynamic updates | `<div aria-live="polite">` |
| `role` | Define purpose | `<div role="alert">` |

### Accessibility Checklist

- Use semantic HTML elements
- All images need `alt` text (empty for decorative: `alt=""`)
- Form inputs need `<label>` elements
- Color contrast: 4.5:1 for text, 3:1 for large text
- Keyboard accessible (Tab, Enter, Escape)
- Focus indicators visible
- Skip navigation link

## Meta Tags & SEO

### Essential Meta Tags

```html
<head>
  <!-- Viewport for responsive -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- Character encoding -->
  <meta charset="UTF-8">

  <!-- Page description (155-160 chars) -->
  <meta name="description" content="Page description for search results">

  <!-- Keywords (less important now) -->
  <meta name="keywords" content="keyword1, keyword2">

  <!-- Robots -->
  <meta name="robots" content="index, follow">
</head>
```

### Open Graph (Facebook, LinkedIn)

```html
<meta property="og:title" content="Page Title">
<meta property="og:description" content="Page description">
<meta property="og:image" content="https://example.com/image.jpg">
<meta property="og:url" content="https://example.com/page">
<meta property="og:type" content="website">
```

### Twitter Cards

```html
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="Page Title">
<meta name="twitter:description" content="Page description">
<meta name="twitter:image" content="https://example.com/image.jpg">
```

### Structured Data (JSON-LD)

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Article Title",
  "author": {
    "@type": "Person",
    "name": "Author Name"
  },
  "datePublished": "2024-01-15"
}
</script>
```

## Web APIs

### localStorage vs sessionStorage

| Feature | localStorage | sessionStorage |
|---------|-------------|----------------|
| Persistence | Until cleared | Until tab closes |
| Capacity | ~5-10 MB | ~5-10 MB |
| Scope | All tabs, same origin | Single tab |
| Use case | User preferences | Form data backup |

```javascript
// Store data
localStorage.setItem('key', 'value')
localStorage.setItem('user', JSON.stringify({ name: 'John' }))

// Retrieve data
const value = localStorage.getItem('key')
const user = JSON.parse(localStorage.getItem('user'))

// Remove data
localStorage.removeItem('key')
localStorage.clear()
```

### Geolocation API

```javascript
// Get current position
navigator.geolocation.getCurrentPosition(
  (position) => {
    const { latitude, longitude } = position.coords
    console.log(latitude, longitude)
  },
  (error) => console.error(error.message),
  { enableHighAccuracy: true, timeout: 5000 }
)

// Watch position changes
const watchId = navigator.geolocation.watchPosition(callback)
navigator.geolocation.clearWatch(watchId)
```

### History API

```javascript
// Add to history
history.pushState({ page: 1 }, 'Title', '/new-url')

// Replace current entry
history.replaceState({ page: 2 }, 'Title', '/another-url')

// Navigate
history.back()
history.forward()
history.go(-2) // Go back 2 pages

// Listen for navigation
window.addEventListener('popstate', (event) => {
  console.log(event.state) // The state object
})
```

## Canvas vs SVG

| Feature | Canvas | SVG |
|---------|--------|-----|
| Type | Raster (pixels) | Vector (shapes) |
| Resolution | Fixed, can pixelate | Scales perfectly |
| DOM access | No | Yes |
| Performance | Better for many objects | Better for few objects |
| Animation | Manual redraw | CSS/SMIL/JS |
| Accessibility | Poor | Good |
| Use cases | Games, photo editing | Icons, charts, logos |

### Canvas Basics

```html
<canvas id="myCanvas" width="400" height="300"></canvas>
```

```javascript
const canvas = document.getElementById('myCanvas')
const ctx = canvas.getContext('2d')

// Draw rectangle
ctx.fillStyle = 'blue'
ctx.fillRect(10, 10, 100, 50)

// Draw circle
ctx.beginPath()
ctx.arc(150, 100, 40, 0, Math.PI * 2)
ctx.fill()

// Draw line
ctx.beginPath()
ctx.moveTo(0, 0)
ctx.lineTo(200, 100)
ctx.stroke()
```

### SVG Basics

```html
<svg width="200" height="200" viewBox="0 0 200 200">
  <!-- Rectangle -->
  <rect x="10" y="10" width="80" height="50" fill="blue" />

  <!-- Circle -->
  <circle cx="150" cy="50" r="40" fill="green" />

  <!-- Line -->
  <line x1="0" y1="100" x2="200" y2="150" stroke="black" />

  <!-- Path -->
  <path d="M10 150 L50 180 L90 150 Z" fill="red" />

  <!-- Text -->
  <text x="100" y="195" text-anchor="middle">SVG Text</text>
</svg>
```

## Quick Reference

### Document Structure

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Page description">
  <title>Page Title</title>
</head>
<body>
  <header><nav>...</nav></header>
  <main>
    <article>
      <section>...</section>
    </article>
    <aside>...</aside>
  </main>
  <footer>...</footer>
</body>
</html>
```

### Common Mistakes to Avoid

- Using `<div>` when semantic elements exist
- Missing `alt` on images
- Missing `<label>` on form inputs
- Missing viewport meta tag
- Using `<br>` for spacing (use CSS)
- Skipping heading levels (h1 â†’ h3)
- Using inline event handlers (`onclick=""`)
