# HTML5 Forms & Validation

Forms are how users interact with your website - signing up, logging in, searching, and submitting data. HTML5 introduced powerful form features that make forms easier to build, more accessible, and more user-friendly without writing JavaScript.

In this lesson, you'll learn how to create accessible forms, use modern input types that optimize mobile keyboards, and implement validation that prevents bad data from reaching your server.

## Form Basics

Every form needs three essential elements:

```html
<form action="/submit" method="POST">
  <label for="username">Username:</label>
  <input type="text" id="username" name="username">

  <button type="submit">Submit</button>
</form>
```

**Key Attributes:**
- `action` - Where to send form data (URL)
- `method` - How to send data (`GET` or `POST`)
- `name` - The field name sent to the server
- `id` - Connects input to label for accessibility

**Important:** Always connect labels to inputs using `for` and `id`. This helps screen readers and allows users to click the label to focus the input.

### Email Input

Validates that the value contains an `@` symbol and looks like an email address.

```html
<form>
  <label for="email">Email:</label>
  <input
    type="email"
    id="email"
    name="email"
    placeholder="you@example.com"
    required
  >
  <button type="submit">Subscribe</button>
</form>
```

**Mobile Benefit:** Shows keyboard with `@` and `.com` keys.

### Telephone Input

Shows numeric keyboard on mobile devices.

```html
<label for="phone">Phone Number:</label>
<input
  type="tel"
  id="phone"
  name="phone"
  placeholder="555-123-4567"
  pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}"
>
```

**Note:** `type="tel"` doesn't validate format - use `pattern` for that.

### URL Input

Validates URL format and shows helpful keyboard on mobile.

```html
<label for="website">Website:</label>
<input
  type="url"
  id="website"
  name="website"
  placeholder="https://example.com"
>
```

### Number Input

Creates a number spinner with increment/decrement buttons.

```html
<label for="age">Age:</label>
<input
  type="number"
  id="age"
  name="age"
  min="18"
  max="120"
  step="1"
  required
>
```

**Attributes:**
- `min` - Minimum value
- `max` - Maximum value
- `step` - Increment amount (can be decimal like `0.01`)

### Date, Time, and DateTime Inputs

HTML5 provides several date/time pickers:

```html
<!-- Date picker -->
<label for="birthday">Birthday:</label>
<input type="date" id="birthday" name="birthday" min="1900-01-01" max="2026-12-31">

<!-- Time picker -->
<label for="appointment">Appointment Time:</label>
<input type="time" id="appointment" name="appointment">

<!-- Date and time combined -->
<label for="meeting">Meeting:</label>
<input type="datetime-local" id="meeting" name="meeting">

<!-- Month picker -->
<label for="month">Select Month:</label>
<input type="month" id="month" name="month">

<!-- Week picker -->
<label for="week">Select Week:</label>
<input type="week" id="week" name="week">
```

**Browser Support:** Modern browsers show native date pickers. Older browsers fall back to text input.

### Color Input

Shows a native color picker.

```html
<label for="theme-color">Theme Color:</label>
<input type="color" id="theme-color" name="color" value="#3b82f6">
```

### Range Input (Slider)

Creates a slider for selecting a value in a range.

```html
<label for="volume">Volume:</label>
<input type="range" id="volume" name="volume" min="0" max="100" value="50">
<output for="volume">50</output>
```

**Tip:** Use `<output>` to display the current value.

### Search Input

Optimized for search queries. Some browsers add a clear button.

```html
<label for="search">Search:</label>
<input type="search" id="search" name="q" placeholder="Search articles...">
```

### Password Input

Masks the entered text for security.

```html
<label for="password">Password:</label>
<input
  type="password"
  id="password"
  name="password"
  minlength="8"
  required
>
```

## HTML5 Validation Attributes

HTML5 provides built-in validation without JavaScript. These attributes prevent form submission if validation fails.

### Required Fields

The `required` attribute makes a field mandatory.

```html
<label for="username">Username:</label>
<input type="text" id="username" name="username" required>

<label for="country">Country:</label>
<select id="country" name="country" required>
  <option value="">Choose...</option>
  <option value="us">United States</option>
  <option value="ca">Canada</option>
</select>
```

**Browser Behavior:** If a required field is empty, the browser shows an error message and prevents submission.

### Minimum and Maximum Length

Control the character count of text inputs.

```html
<label for="username">Username (3-20 characters):</label>
<input
  type="text"
  id="username"
  name="username"
  minlength="3"
  maxlength="20"
  required
>

<label for="bio">Bio (max 500 characters):</label>
<textarea
  id="bio"
  name="bio"
  maxlength="500"
></textarea>
```

**Difference:**
- `minlength` - Prevents submission if too short
- `maxlength` - Prevents typing beyond limit (hard stop)

### Pattern Validation (Regular Expressions)

Use the `pattern` attribute to validate against a regular expression.

```html
<!-- US ZIP code -->
<label for="zip">ZIP Code:</label>
<input
  type="text"
  id="zip"
  name="zip"
  pattern="[0-9]{5}"
  title="5-digit ZIP code"
  placeholder="12345"
>

<!-- Phone number -->
<label for="phone">Phone:</label>
<input
  type="tel"
  id="phone"
  name="phone"
  pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}"
  title="Format: 555-555-5555"
  placeholder="555-555-5555"
>

<!-- Username (alphanumeric, 3-16 chars) -->
<label for="username">Username:</label>
<input
  type="text"
  id="username"
  name="username"
  pattern="[a-zA-Z0-9]{3,16}"
  title="3-16 alphanumeric characters"
>
```

**Important:** Always include the `title` attribute to explain the required format.

### Min and Max for Numbers and Dates

```html
<!-- Age between 18 and 120 -->
<label for="age">Age:</label>
<input
  type="number"
  id="age"
  name="age"
  min="18"
  max="120"
  required
>

<!-- Future dates only -->
<label for="appointment">Appointment Date:</label>
<input
  type="date"
  id="appointment"
  name="appointment"
  min="2026-01-15"
>
```

### Step Attribute

Control the granularity of numeric inputs.

```html
<!-- Integers only -->
<input type="number" step="1">

<!-- Decimals (price) -->
<input type="number" step="0.01" min="0">

<!-- Multiples of 5 -->
<input type="number" step="5" min="0" max="100">
```

## Disabling Browser Validation

Sometimes you want to handle validation with JavaScript instead. Use the `novalidate` attribute.

```html
<form action="/submit" method="POST" novalidate>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>
  <button type="submit">Submit</button>
</form>
```

**When to Use:**
- Custom validation with JavaScript libraries
- Real-time validation as user types
- Complex validation logic
- Better error message styling

Here's a registration form using HTML5 features:

```html
<form action="/register" method="POST">
  <h2>Create Account</h2>

  <!-- Text Input -->
  <label for="fullname">Full Name:</label>
  <input
    type="text"
    id="fullname"
    name="fullname"
    placeholder="John Doe"
    required
    minlength="2"
  >

  <!-- Email Input -->
  <label for="email">Email:</label>
  <input
    type="email"
    id="email"
    name="email"
    placeholder="you@example.com"
    required
  >

  <!-- Password -->
  <label for="password">Password:</label>
  <input
    type="password"
    id="password"
    name="password"
    minlength="8"
    pattern="(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{8,}"
    title="Must contain at least one number, one uppercase and lowercase letter, and at least 8 characters"
    required
  >

  <!-- Phone Number -->
  <label for="phone">Phone Number:</label>
  <input
    type="tel"
    id="phone"
    name="phone"
    pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}"
    placeholder="555-555-5555"
  >

  <!-- Date of Birth -->
  <label for="dob">Date of Birth:</label>
  <input
    type="date"
    id="dob"
    name="dob"
    max="2008-12-31"
    required
  >

  <!-- Select Dropdown -->
  <label for="country">Country:</label>
  <select id="country" name="country" required>
    <option value="">Select your country</option>
    <option value="us">United States</option>
    <option value="ca">Canada</option>
    <option value="uk">United Kingdom</option>
  </select>

  <!-- Checkbox -->
  <label>
    <input type="checkbox" name="terms" required>
    I agree to the Terms of Service
  </label>

  <!-- Submit Button -->
  <button type="submit">Create Account</button>
</form>
```

### Using Fieldset and Legend

Group related form fields with `<fieldset>` and `<legend>`:

```html
<form>
  <fieldset>
    <legend>Personal Information</legend>
    <label for="fname">First Name:</label>
    <input type="text" id="fname" name="fname">

    <label for="lname">Last Name:</label>
    <input type="text" id="lname" name="lname">
  </fieldset>

  <fieldset>
    <legend>Contact Information</legend>
    <label for="email">Email:</label>
    <input type="email" id="email" name="email">

    <label for="phone">Phone:</label>
    <input type="tel" id="phone" name="phone">
  </fieldset>

  <button type="submit">Submit</button>
</form>
```

### Radio Buttons and Checkboxes

```html
<!-- Radio Buttons (single choice) -->
<fieldset>

  </label>

  <label>

    Pro - $9.99/month
  </label>

  <label>
    <input type="radio" name="plan" value="enterprise">

</fieldset>
```

## Common Mistakes to Avoid

1. **Missing labels**
   - Every input needs a `<label>` connected via `for` and `id`
   - Don't rely on placeholder text alone

2. **Using placeholder as a label**
   - Placeholders disappear when you start typing
   - Always use proper `<label>` elements

3. **Not providing error context**
   - Use the `title` attribute on `pattern` inputs to explain the format
   - Consider custom error messages with JavaScript

4. **Incorrect input types**
   - Using `type="text"` for everything
   - Use specific types (email, tel, url, number, date) for better UX

5. **Client-side validation only**
   - Always validate on the server too
   - Client-side validation is for UX, not security

## Key Takeaways

- **HTML5 input types** improve user experience and show appropriate mobile keyboards
- **Validation attributes** (required, minlength, maxlength, pattern, min, max) prevent invalid submissions
- **Always connect labels to inputs** using `for` and `id` for accessibility
- **Use `pattern` with `title`** to explain the required format to users
- **The `novalidate` attribute** disables browser validation for custom JavaScript validation
- **Group related fields** with `<fieldset>` and `<legend>`
- **Email, tel, and url types** provide automatic validation
- **Date and time inputs** show native pickers in modern browsers
- **Server-side validation is mandatory** - never trust client-side validation alone

HTML5 forms are powerful, accessible, and user-friendly. Use the right input types and validation attributes to create forms that guide users toward success while preventing errors.
