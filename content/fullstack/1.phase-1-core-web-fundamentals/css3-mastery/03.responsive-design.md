# Responsive Design (media queries, mobile-first)

Today, people browse websites on phones, tablets, laptops, and large monitors. Your site needs to look good on all of them. Responsive design is how you make that happen.

In this lesson, you'll learn the mobile-first approach, master media queries, and use responsive units to create flexible layouts that work everywhere.

## What Is Responsive Design?

**Responsive design** means your website adapts to different screen sizes and devices. The layout, text size, images, and navigation all adjust automatically.

**Three Core Principles:**

1. **Fluid layouts** - Use flexible units instead of fixed pixels
2. **Flexible images** - Images scale with their containers
3. **Media queries** - Apply different styles based on screen size

## Mobile-First Approach

The **mobile-first** approach means you design for small screens first, then add styles for larger screens.

**Why Mobile-First?**

| Benefit | Explanation |
|---------|-------------|
| **Better performance** | Mobile devices load only essential CSS |
| **Progressive enhancement** | Add features as screen space allows |
| **Forces prioritization** | You focus on what's truly important |
| **Easier to scale up** | Adding features is easier than removing them |

### Mobile-First vs Desktop-First

**Desktop-First (Old Way):**

```css
/* Base styles for desktop */
.container {
  width: 1200px;
  padding: 3rem;
}

/* Scale down for mobile */
@media (max-width: 768px) {
  .container {
    width: 100%;
    padding: 1rem;
  }
}
```

**Mobile-First (Modern Way):**

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 1rem;
}

/* Scale up for larger screens */
@media (min-width: 768px) {
  .container {
    width: 1200px;
    padding: 3rem;
  }
}
```

Notice how mobile-first uses `min-width` (screens **at least** this wide), while desktop-first uses `max-width` (screens **no wider than** this).

## Media Queries

**Media queries** let you apply CSS based on device characteristics like screen width.

### Basic Syntax

```css
@media (condition) {
  /* CSS rules */
}
```

**Example - Tablet and Up:**

```css
/* Mobile styles (base) */
.nav {
  flex-direction: column;
}

/* Tablet and larger */
@media (min-width: 768px) {
  .nav {
    flex-direction: row;
  }
}
```

### Common Breakpoints

These are standard screen sizes where designs often need to change:

| Breakpoint | Size | Device Type |
|------------|------|-------------|
| Extra Small | 0-479px | Phones (portrait) |
| Small | 480-767px | Phones (landscape) |
| Medium | 768-1023px | Tablets |
| Large | 1024-1279px | Small laptops |
| Extra Large | 1280px+ | Desktops |

**Mobile-First Media Queries:**

```css
/* Mobile (base) - no media query needed */

/* Small phones landscape */
@media (min-width: 480px) {
  /* Styles */
}

/* Tablets */
@media (min-width: 768px) {
  /* Styles */
}

/* Laptops */
@media (min-width: 1024px) {
  /* Styles */
}

/* Desktops */
@media (min-width: 1280px) {
  /* Styles */
}
```

### Multiple Conditions

Combine conditions with `and`:

```css
/* Tablets in landscape mode */
@media (min-width: 768px) and (max-width: 1023px) and (orientation: landscape) {
  /* Styles for tablets in landscape */
}
```

### Media Types

Target specific media types:

```css
/* Only for screens (default) */
@media screen and (min-width: 768px) {
  /* Styles */
}

/* Only for print */
@media print {
  /* Hide navigation when printing */
  nav, footer {
    display: none;
  }
}
```

## Responsive Units

Using the right units is key to responsive design.

### rem - Root Em

**rem** is relative to the root element's font size (usually 16px):

```css
html {
  font-size: 16px; /* 1rem = 16px */
}

.heading {
  font-size: 2rem;   /* 32px */
  margin-bottom: 1rem; /* 16px */
}

.small-text {
  font-size: 0.875rem; /* 14px */
}
```

**Why use rem?**
- Consistent sizing across the site
- Users can change text size in browser settings
- Easy to calculate (16px base makes math simple)

### em - Relative to Parent

**em** is relative to the parent element's font size:

```css
.card {
  font-size: 18px;
}

.card .title {
  font-size: 1.5em; /* 27px (18 * 1.5) */
}

.card .description {
  font-size: 1em; /* 18px (same as parent) */
}
```

**When to use em:**
- Component-level spacing that scales with text
- Nested components that should scale together

**Warning:** em compounds through nesting, which can be confusing. rem is usually safer.

### Percentages (%)

**Percentages** are relative to the parent element's size:

```css
.container {
  width: 100%; /* Full width of parent */
}

.sidebar {
  width: 25%; /* 25% of parent width */
}

.content {
  width: 75%; /* 75% of parent width */
}
```

**Common uses:**
- Widths (width: 50%)
- Heights (height: 100%)
- Positioning (top: 50%)

### vw/vh - Viewport Units

**vw** (viewport width) and **vh** (viewport height) are relative to the viewport:

- `1vw` = 1% of viewport width
- `1vh` = 1% of viewport height
- `100vw` = full viewport width
- `100vh` = full viewport height

```css
/* Full-screen hero section */
.hero {
  width: 100vw;
  height: 100vh;
}

/* Responsive text */
.heading {
  font-size: 5vw; /* Scales with viewport */
}
```

**Warning:** Be careful with viewport units on mobile - the address bar can cause issues. Use `min-height: 100vh` instead of `height: 100vh`.

### Unit Comparison

| Unit | Relative To | Best For |
|------|-------------|----------|
| px | Fixed | Borders, shadows |
| rem | Root font size | Spacing, font sizes |
| em | Parent font size | Component spacing |
| % | Parent size | Widths, heights |
| vw/vh | Viewport | Full-screen sections |

  <div class="product">Product 4</div>

/* Mobile - 1 column */
.products {
  display: grid;
  grid-template-columns: 1fr;
  gap: 1rem;

.product {
  padding: 1.5rem;
  background: white;
  border-radius: 8px;

  .products {

    gap: 2rem;
    padding: 3rem;
  }

```css
/* Mobile - smaller, tighter spacing */

  font-size: 16px;
  line-height: 1.5;
}

h1 {
  font-size: 2rem;   /* 32px */
  margin-bottom: 1rem;
}

h2 {
  font-size: 1.5rem; /* 24px */
  margin-bottom: 0.75rem;
}

p {
  margin-bottom: 1rem;
}

/* Tablet - slightly larger */
@media (min-width: 768px) {
  body {
    font-size: 18px;
  }

  h1 {
    font-size: 2.5rem; /* 45px */
  }

  h2 {
    font-size: 2rem; /* 36px */
  }
}

/* Desktop - more generous spacing */
@media (min-width: 1024px) {
  h1 {
    font-size: 3rem; /* 48px */
    margin-bottom: 1.5rem;
  }

  h2 {
    font-size: 2.25rem; /* 36px */
    margin-bottom: 1rem;
  }
}
```

```css

img {

  .hero-image {

    margin: 0 auto;
  }

```

## The Viewport Meta Tag

For responsive design to work on mobile devices, you need this meta tag in your HTML `<head>`:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

**What it does:**
- `width=device-width` - Sets viewport width to device width
- `initial-scale=1.0` - Sets initial zoom level to 100%

Without this tag, mobile browsers render your page at desktop width and zoom out, breaking your responsive design.

## Best Practices

1. **Start with mobile, enhance for larger screens**
   - Write base styles for mobile
   - Add complexity with min-width media queries

2. **Use relative units for flexibility**
   - rem for fonts and spacing
   - % or fr for widths
   - Avoid fixed pixel widths

3. **Test on real devices**
   - Browser dev tools are good, but not perfect
   - Test on actual phones and tablets

4. **Set breakpoints based on content, not devices**
   - Don't target specific device sizes
   - Let your content determine where to break

5. **Use Flexbox and Grid for layouts**
   - They adapt naturally to screen size
   - Less need for complex media queries

6. **Optimize images for different screens**
   - Use responsive images (srcset)
   - Serve smaller images to mobile devices

## Common Mistakes to Avoid

1. **Forgetting the viewport meta tag**
   - Mobile browsers need this to render properly
   - Always include it in `<head>`

2. **Using fixed pixel widths**
   - `width: 500px` doesn't adapt
   - Use `max-width: 500px` or percentages

3. **Too many breakpoints**
   - More breakpoints = more code to maintain
   - Start with 2-3 breakpoints, add only if needed

4. **Not testing on real devices**
   - Desktop dev tools don't catch everything
   - Test on actual phones and tablets

5. **Ignoring touch targets on mobile**
   - Make buttons at least 44x44px for easy tapping
   - Add enough spacing between clickable elements

## Responsive Design Checklist

Before launching a responsive site, check:

- [ ] Viewport meta tag is present
- [ ] Mobile navigation works (hamburger menu)
- [ ] Text is readable without zooming
- [ ] Buttons and links are easy to tap (44x44px minimum)
- [ ] Images scale properly
- [ ] No horizontal scrolling on mobile
- [ ] Forms are easy to fill on mobile
- [ ] Tested on multiple devices and screen sizes

## Key Takeaways

- **Responsive design** adapts layouts to different screen sizes
- **Mobile-first** means designing for small screens first, then enhancing for larger screens
- **Media queries** apply styles based on screen width using @media
- Use **min-width** for mobile-first (add styles as screens grow)
- Use **max-width** for desktop-first (remove styles as screens shrink)
- **rem** is best for spacing and fonts (relative to root)
- **%** is best for widths (relative to parent)
- **vw/vh** is for viewport-relative sizing
- Always include the **viewport meta tag** in HTML
- Test on real devices, not just browser dev tools

Responsive design is essential for modern web development. Master media queries and responsive units, and your sites will work beautifully on any device.
