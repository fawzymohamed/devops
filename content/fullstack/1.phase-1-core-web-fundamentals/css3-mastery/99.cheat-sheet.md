# CSS3 Mastery Cheat Sheet

Quick reference for modern CSS including Flexbox, Grid, responsive design, variables, animations, and architecture patterns.

---

## Flexbox

**Enable Flexbox:** `display: flex;`

### Container Properties

| Property | Values | Description |
|----------|--------|-------------|
| `flex-direction` | `row`, `row-reverse`, `column`, `column-reverse` | Main axis direction |
| `flex-wrap` | `nowrap`, `wrap`, `wrap-reverse` | Allow wrapping |
| `justify-content` | `flex-start`, `flex-end`, `center`, `space-between`, `space-around`, `space-evenly` | Main axis alignment |
| `align-items` | `stretch`, `flex-start`, `flex-end`, `center`, `baseline` | Cross axis alignment |
| `align-content` | `stretch`, `flex-start`, `flex-end`, `center`, `space-between`, `space-around` | Multi-line alignment |
| `gap` | `10px`, `1rem 2rem` | Space between items |

### Item Properties

| Property | Values | Description |
|----------|--------|-------------|
| `flex-grow` | `0`, `1`, etc. | Grow ratio |
| `flex-shrink` | `0`, `1`, etc. | Shrink ratio |
| `flex-basis` | `auto`, `100px`, `50%` | Initial size |
| `flex` | `1`, `1 1 auto`, `0 0 200px` | Shorthand |
| `align-self` | `auto`, `flex-start`, `flex-end`, `center` | Override align-items |
| `order` | `0`, `-1`, `1` | Display order |

### Common Flexbox Patterns

```css
/* Center everything */
.center {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* Space between with gap */
.nav {
  display: flex;
  justify-content: space-between;
  gap: 1rem;
}

/* Equal width columns */
.columns {
  display: flex;
}
.columns > * {
  flex: 1;
}

/* Sticky footer */
.page {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}
.page main {
  flex: 1;
}
```

## CSS Grid

**Enable Grid:** `display: grid;`

### Container Properties

| Property | Example | Description |
|----------|---------|-------------|
| `grid-template-columns` | `1fr 1fr 1fr`, `repeat(3, 1fr)` | Column sizes |
| `grid-template-rows` | `auto 1fr auto` | Row sizes |
| `gap` | `20px`, `1rem 2rem` | Row and column gaps |
| `grid-auto-rows` | `minmax(100px, auto)` | Implicit row size |
| `grid-auto-flow` | `row`, `column`, `dense` | Auto placement |

### Item Properties

| Property | Example | Description |
|----------|---------|-------------|
| `grid-column` | `1 / 3`, `span 2` | Column span |
| `grid-row` | `1 / 3`, `span 2` | Row span |
| `grid-area` | `header`, `1 / 1 / 2 / 3` | Named area or shorthand |
| `justify-self` | `start`, `end`, `center`, `stretch` | Horizontal alignment |
| `align-self` | `start`, `end`, `center`, `stretch` | Vertical alignment |

### Useful Grid Functions

| Function | Example | Description |
|----------|---------|-------------|
| `repeat()` | `repeat(3, 1fr)` | Repeat pattern |
| `minmax()` | `minmax(200px, 1fr)` | Min and max size |
| `auto-fit` | `repeat(auto-fit, minmax(200px, 1fr))` | Fit items, collapse empty |
| `auto-fill` | `repeat(auto-fill, minmax(200px, 1fr))` | Fill space, keep empty |

### Common Grid Patterns

```css
/* Responsive card grid */
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
  gap: 1.5rem;
}

/* Holy grail layout */
.page {
  display: grid;
  grid-template-areas:
    "header header header"
    "nav    main   aside"
    "footer footer footer";
  grid-template-columns: 200px 1fr 200px;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}
.header { grid-area: header; }
.nav    { grid-area: nav; }
.main   { grid-area: main; }
.aside  { grid-area: aside; }
.footer { grid-area: footer; }

/* Two-column with sidebar */
.layout {
  display: grid;
  grid-template-columns: 1fr 300px;
  gap: 2rem;
}
```

## Responsive Design

### Mobile-First Media Queries

```css
/* Base styles (mobile) */
.container {
  padding: 1rem;
}

/* Tablet (768px+) */
@media (min-width: 768px) {
  .container {
    padding: 2rem;
  }
}

/* Desktop (1024px+) */
@media (min-width: 1024px) {
  .container {
    max-width: 1200px;
    margin: 0 auto;
  }
}
```

### Common Breakpoints

| Breakpoint | Size | Device |
|------------|------|--------|
| sm | 480px | Phones landscape |
| md | 768px | Tablets |
| lg | 1024px | Small laptops |
| xl | 1280px | Desktops |

### Responsive Units

| Unit | Relative To | Best For |
|------|-------------|----------|
| `rem` | Root font size (16px) | Fonts, spacing |
| `em` | Parent font size | Component spacing |
| `%` | Parent size | Widths |
| `vw` | Viewport width | Full-width elements |
| `vh` | Viewport height | Full-height sections |

### Essential Meta Tag

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

## CSS Variables

### Defining Variables

```css
:root {
  /* Colors */
  --color-primary: #3b82f6;
  --color-secondary: #10b981;
  --color-danger: #ef4444;

  /* Spacing */
  --spacing-sm: 0.5rem;
  --spacing-md: 1rem;
  --spacing-lg: 2rem;

  /* Typography */
  --font-sans: 'Inter', sans-serif;
  --text-base: 1rem;
}
```

### Using Variables

```css
.button {
  background: var(--color-primary);
  padding: var(--spacing-sm) var(--spacing-md);
  font-family: var(--font-sans);
}

/* With fallback */
.element {
  color: var(--text-color, black);
}
```

### Theming with Variables

```css
:root {
  --bg-primary: white;
  --text-primary: #111827;
}

.dark-theme {
  --bg-primary: #111827;
  --text-primary: #f9fafb;
}

body {
  background: var(--bg-primary);
  color: var(--text-primary);
}
```

### JavaScript Integration

```javascript
// Get variable value
getComputedStyle(document.documentElement)
  .getPropertyValue('--color-primary')

// Set variable value
document.documentElement.style
  .setProperty('--color-primary', '#ef4444')
```

## Animations & Transitions

### Transitions

```css
.button {
  background: #3b82f6;
  transition: background 0.3s ease;
}

.button:hover {
  background: #2563eb;
}

/* Multiple properties */
.card {
  transition: transform 0.3s ease, box-shadow 0.3s ease;
}

/* All properties */
.element {
  transition: all 0.3s ease;
}
```

### Transition Timing Functions

| Function | Description |
|----------|-------------|
| `ease` | Slow start, fast middle, slow end |
| `ease-in` | Slow start |
| `ease-out` | Slow end |
| `ease-in-out` | Slow start and end |
| `linear` | Constant speed |
| `cubic-bezier(n,n,n,n)` | Custom curve |

### Keyframe Animations

```css
@keyframes fadeIn {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.element {
  animation: fadeIn 0.5s ease forwards;
}

/* Multiple keyframes */
@keyframes bounce {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-20px); }
}
```

### Animation Properties

| Property | Example | Description |
|----------|---------|-------------|
| `animation-name` | `fadeIn` | Keyframe name |
| `animation-duration` | `0.5s`, `500ms` | How long |
| `animation-timing-function` | `ease`, `linear` | Speed curve |
| `animation-delay` | `0.2s` | Wait before start |
| `animation-iteration-count` | `1`, `infinite` | Repeat count |
| `animation-direction` | `normal`, `reverse`, `alternate` | Direction |
| `animation-fill-mode` | `forwards`, `backwards`, `both` | State when done |

### Performance Tips

- Animate `transform` and `opacity` (GPU accelerated)
- Avoid animating `width`, `height`, `top`, `left`
- Use `will-change` sparingly for heavy animations

## Pseudo-classes & Pseudo-elements

### Common Pseudo-classes

| Selector | Description |
|----------|-------------|
| `:hover` | Mouse over element |
| `:focus` | Element has focus |
| `:active` | Being clicked |
| `:visited` | Visited link |
| `:first-child` | First child of parent |
| `:last-child` | Last child of parent |
| `:nth-child(n)` | Nth child (1-indexed) |
| `:nth-child(odd)` | Odd children (1, 3, 5...) |
| `:nth-child(even)` | Even children (2, 4, 6...) |
| `:nth-child(3n)` | Every 3rd (3, 6, 9...) |
| `:not(.class)` | Negation |
| `:disabled` | Disabled form elements |
| `:checked` | Checked inputs |
| `:focus-visible` | Focus for keyboard only |

### Pseudo-elements

| Selector | Description |
|----------|-------------|
| `::before` | Insert before content |
| `::after` | Insert after content |
| `::first-letter` | First letter |
| `::first-line` | First line |
| `::selection` | Selected text |
| `::placeholder` | Input placeholder |

### Pseudo-element Examples

```css
/* Required field indicator */
.required::after {
  content: " *";
  color: red;
}

/* Custom bullets */
li::before {
  content: "✓";
  color: green;
  margin-right: 0.5rem;
}

/* Tooltip */
.tooltip::after {
  content: attr(data-tooltip);
  position: absolute;
  /* positioning styles */
}
```

> **Note:** `::before` and `::after` require the `content` property, even if empty (`content: ""`).

## Specificity & Cascade

### Specificity Calculation

| Selector Type | Points | Example |
|---------------|--------|---------|
| Inline styles | 1-0-0-0 | `style="..."` |
| ID selectors | 1-0-0 | `#header` |
| Classes, attributes, pseudo-classes | 0-1-0 | `.menu`, `[type]`, `:hover` |
| Elements, pseudo-elements | 0-0-1 | `div`, `::before` |

### Specificity Examples

```css
/* 0-0-1 */
p { }

/* 0-1-0 */
.card { }

/* 1-0-0 */
#header { }

/* 0-1-1 */
div.card { }

/* 1-1-1 */
#nav .menu li { }
```

### Cascade Priority (Low to High)

1. Browser defaults
2. External stylesheets
3. Internal styles (`<style>`)
4. Inline styles (`style=""`)
5. `!important` rules

### When Specificity is Equal

**Last rule wins** (source order matters)

```css
.button { color: blue; }
.button { color: red; }  /* This wins */
```

### Inherited Properties

| Inherited | Not Inherited |
|-----------|---------------|
| `color` | `width`, `height` |
| `font-*` | `margin`, `padding` |
| `line-height` | `border` |
| `text-*` | `background` |
| `cursor` | `display`, `position` |

### Special Values

```css
.element {
  color: inherit;   /* Use parent's value */
  color: initial;   /* CSS default */
  color: unset;     /* inherit if inherited, initial if not */
}
```

### !important Best Practices

```css
/* AVOID - makes CSS hard to maintain */
.button { background: blue !important; }

/* OK - utility classes */
.hidden { display: none !important; }

/* OK - override third-party styles */
.third-party { margin: 0 !important; }
```

## CSS Architecture

### BEM Naming Convention

**Block__Element--Modifier**

```css
/* Block */
.card { }

/* Element */
.card__title { }
.card__body { }
.card__button { }

/* Modifier */
.card--featured { }
.card__button--primary { }
.card__button--disabled { }
```

### BEM Example

```html
<article class="card card--featured">
  <h2 class="card__title">Title</h2>
  <div class="card__body">Content</div>
  <button class="card__button card__button--primary">Click</button>
</article>
```

### SMACSS Categories

| Category | Purpose | Example |
|----------|---------|---------|
| Base | Default styles | `body`, `a`, `h1` |
| Layout | Page structure | `.l-header`, `.l-sidebar` |
| Module | Reusable components | `.card`, `.button` |
| State | Temporary states | `.is-active`, `.is-hidden` |
| Theme | Visual variations | `.theme-dark` |

### Architecture Best Practices

- Use low specificity selectors
- Avoid `!important`
- Keep selectors flat (avoid deep nesting)
- Use classes, not IDs for styling
- One component per file
- Document complex styles

## Quick Reference

### Box Model

```css
.element {
  /* Content → Padding → Border → Margin */
  box-sizing: border-box; /* Include padding/border in width */
  width: 100%;
  padding: 1rem;
  border: 1px solid #ccc;
  margin: 1rem 0;
}
```

### Centering Methods

```css
/* Flexbox (recommended) */
.parent {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* Grid */
.parent {
  display: grid;
  place-items: center;
}

/* Absolute positioning */
.child {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
```

## Interview Quick Hits

| Question | Answer |
|----------|--------|
| What's the difference between Flexbox and Grid? | Flexbox is one-dimensional (row OR column); Grid is two-dimensional (rows AND columns) |
| How does CSS specificity work? | Inline styles > IDs > Classes/attributes > Elements; higher specificity wins, last rule wins if equal |
| What is the box model? | Content → Padding → Border → Margin; use `box-sizing: border-box` to include padding/border in width |
| What is BEM? | Block Element Modifier - a naming convention for CSS classes: `.block__element--modifier` |
| What properties should you animate for best performance? | `transform` and `opacity` - they're GPU accelerated and don't trigger layout |
| What's the difference between `rem` and `em`? | `rem` is relative to root font size; `em` is relative to parent element's font size |

### Common Mistakes to Avoid

- Using `px` for fonts (use `rem`)
- Fixed widths instead of `max-width`
- Forgetting viewport meta tag
- Removing `:focus` styles without alternative
- Using `float` for layout (use Flexbox/Grid)
- Over-qualifying selectors (`.nav ul li a`)
