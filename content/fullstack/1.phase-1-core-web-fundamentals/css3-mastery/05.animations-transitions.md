# Animations & Transitions (keyframes, easing, transform)

Motion brings interfaces to life. Smooth transitions and animations make your site feel professional and responsive. CSS gives you powerful tools to create stunning effects without JavaScript.

In this lesson, you'll learn the difference between transitions and animations, master keyframe animations, use transforms efficiently, and understand performance best practices.

## Transitions vs Animations

**CSS Transitions** - Smooth changes between two states (A â†’ B)
- Need a trigger (hover, focus, class change)
- Simple, one-time changes
- Perfect for hover effects, show/hide

**CSS Animations** - Complex, multi-step sequences
- Run automatically when applied
- Can loop infinitely
- Can have multiple keyframes
- Perfect for loading spinners, attention-grabbers

## CSS Transitions

Transitions smoothly animate CSS property changes.

### Basic Syntax

```css
.element {
  transition: property duration timing-function delay;
}
```

**Example - Button Hover:**

```css
.button {
  background: #3b82f6;
  color: white;
  padding: 0.75rem 1.5rem;
  border: none;
  border-radius: 6px;
  transition: background 0.3s ease;
}

.button:hover {
  background: #2563eb;
}
```

The background color smoothly changes over 0.3 seconds when you hover.

### Transition Properties

1. **transition-property** - What to animate

```css
/* Single property */
transition-property: background-color;

/* Multiple properties */
transition-property: background-color, transform, opacity;

/* All animatable properties */
transition-property: all;
```

2. **transition-duration** - How long

```css
transition-duration: 0.3s;    /* 300 milliseconds */
transition-duration: 500ms;   /* Same as 0.5s */
```

3. **transition-timing-function** - Speed curve

| Timing Function | Effect | Use Case |
|-----------------|--------|----------|
| `linear` | Constant speed | Progress bars |
| `ease` | Slow start and end (default) | General use |
| `ease-in` | Slow start | Elements appearing |
| `ease-out` | Slow end | Elements disappearing |
| `ease-in-out` | Slow start and end | Smooth, natural motion |
| `cubic-bezier(x1,y1,x2,y2)` | Custom curve | Fine control |

```css
/* Default timing */
transition: all 0.3s ease;

/* Smooth, natural feel */
transition: all 0.3s ease-in-out;

/* Custom curve */
transition: all 0.3s cubic-bezier(0.68, -0.55, 0.265, 1.55);
```

4. **transition-delay** - Wait before starting

```css
transition-delay: 0.1s; /* Wait 100ms before starting */
```

### Shorthand Syntax

```css
/* property duration timing-function delay */
transition: background 0.3s ease 0s;

/* Multiple transitions */
transition: background 0.3s ease,
            transform 0.3s ease-out,
            opacity 0.2s linear;
```

### Practical Transition Examples

**Example 1 - Card Hover:**

```html
<div class="card">
  <img src="product.jpg" alt="Product">
  <h3>Product Name</h3>
  <p>$29.99</p>
</div>
```

```css

  padding: 1.5rem;
  background: white;
  border-radius: 12px;
  box-shadow: 0 2px 8px rgba(0,0,0,0.1);
  transition: transform 0.3s ease, box-shadow 0.3s ease;
  cursor: pointer;
}

.card:hover {
  transform: translateY(-8px);
  box-shadow: 0 12px 24px rgba(0,0,0,0.15);
}

.card img {
  width: 100%;
  border-radius: 8px;
  transition: transform 0.3s ease;
}

.card:hover img {
  transform: scale(1.05);
}
```

**Example 2 - Smooth Menu:**

```css
.nav-link {
  color: #6b7280;
  text-decoration: none;
  padding: 0.5rem 1rem;
  border-bottom: 3px solid transparent;
  transition: color 0.2s ease, border-color 0.2s ease;
}

.nav-link:hover {
  color: #3b82f6;
  border-bottom-color: #3b82f6;
}
```

## CSS Transforms

The `transform` property lets you modify elements in 2D or 3D space without affecting document layout.

### Transform Functions

#### 1. translate - Move

```css
/* Move right 50px, down 100px */
transform: translate(50px, 100px);

/* Move only horizontally */
transform: translateX(50px);

/* Move only vertically */
transform: translateY(100px);

/* Percentage of element's size */
transform: translate(50%, 50%);
```

#### 2. rotate - Rotate

```css
/* Rotate 45 degrees clockwise */
transform: rotate(45deg);

/* Rotate counterclockwise */
transform: rotate(-45deg);

/* Full rotation */
transform: rotate(360deg);
```

#### 3. scale - Resize

```css
/* Double size */
transform: scale(2);

/* Half size */
transform: scale(0.5);

/* Different horizontal and vertical */
transform: scale(1.5, 0.8);

/* Scale only width */
transform: scaleX(1.5);

/* Scale only height */
transform: scaleY(0.8);
```

#### 4. skew - Slant

```css
/* Skew horizontally */
transform: skewX(20deg);

/* Skew vertically */
transform: skewY(20deg);

/* Skew both */
transform: skew(20deg, 10deg);
```

### Combining Transforms

You can combine multiple transforms:

```css
/* Transforms are applied left to right */
transform: translate(50px, 0) rotate(45deg) scale(1.2);
```

**Order matters:**
```css
/* These produce different results */
transform: rotate(45deg) translate(100px, 0);
transform: translate(100px, 0) rotate(45deg);
```

### Transform Origin

By default, transforms happen around the element's center. Change this with `transform-origin`:

```css
/* Default - center */
transform-origin: center center;

/* Top left corner */
transform-origin: top left;

/* Specific coordinates */
transform-origin: 20px 30px;

/* Percentages */
transform-origin: 50% 50%; /* Center (default) */
```

**Example - Card Flip:**

```css
.card {
  width: 300px;
  height: 200px;
  position: relative;
  transform-style: preserve-3d;
  transition: transform 0.6s;
}

.card:hover {
  transform: rotateY(180deg);
}

.card-front, .card-back {
  position: absolute;
  width: 100%;
  height: 100%;
  backface-visibility: hidden;
}

.card-back {
  transform: rotateY(180deg);
}
```

## CSS Animations with @keyframes

Animations let you create multi-step sequences.

### Creating Keyframes

```css
@keyframes animation-name {
  from {
    /* Starting state */
  }
  to {
    /* Ending state */
  }
}

/* Or with percentages for more control */
@keyframes animation-name {
  0% {
    /* Start */
  }
  50% {
    /* Middle */
  }
  100% {
    /* End */
  }
}
```

### Applying Animations

```css
.element {
  animation: animation-name duration timing-function delay iteration-count direction;
}
```

**Example - Fade In:**

```css
@keyframes fadeIn {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}

.fade-in {
  animation: fadeIn 0.5s ease;
}
```

### Animation Properties

1. **animation-name** - Which keyframes to use

```css
animation-name: fadeIn;
```

2. **animation-duration** - How long

```css
animation-duration: 2s;
```

3. **animation-timing-function** - Speed curve

```css
animation-timing-function: ease-in-out;
```

4. **animation-delay** - Wait before starting

```css
animation-delay: 0.5s;
```

5. **animation-iteration-count** - How many times

```css
animation-iteration-count: 1;       /* Once (default) */
animation-iteration-count: 3;       /* Three times */
animation-iteration-count: infinite; /* Forever */
```

6. **animation-direction** - Play forward or backward

```css
animation-direction: normal;           /* Forward (default) */
animation-direction: reverse;          /* Backward */
animation-direction: alternate;        /* Forward, then backward */
animation-direction: alternate-reverse; /* Backward, then forward */
```

7. **animation-fill-mode** - Styles before/after animation

```css
animation-fill-mode: none;      /* Default - no persistence */
animation-fill-mode: forwards;  /* Keep final state */
animation-fill-mode: backwards; /* Apply first keyframe immediately */
animation-fill-mode: both;      /* Both forwards and backwards */
```

### Practical Animation Examples

**Example 1 - Loading Spinner:**

```html
<div class="spinner"></div>
```

```css
.spinner {
  width: 40px;
  height: 40px;
  border: 4px solid #f3f4f6;
  border-top-color: #3b82f6;
  border-radius: 50%;
  animation: spin 1s linear infinite;
}

@keyframes spin {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

**Example 2 - Pulse Effect:**

```css
.notification-badge {
  width: 20px;
  height: 20px;
  background: #ef4444;
  border-radius: 50%;
  animation: pulse 2s ease infinite;
}

@keyframes pulse {
  0%, 100% {
    transform: scale(1);
    opacity: 1;
  }
  50% {
    transform: scale(1.2);
    opacity: 0.8;
  }
}
```

**Example 3 - Slide In from Left:**

```css
.slide-in {
  animation: slideInLeft 0.5s ease forwards;
}

@keyframes slideInLeft {
  from {
    transform: translateX(-100%);
    opacity: 0;
  }
  to {
    transform: translateX(0);
    opacity: 1;
  }
}
```

**Example 4 - Bounce:**

```css
.bounce {
  animation: bounce 1s ease infinite;
}

@keyframes bounce {
  0%, 20%, 50%, 80%, 100% {
    transform: translateY(0);
  }
  40% {
    transform: translateY(-30px);
  }
  60% {
    transform: translateY(-15px);
  }
}
```

**Example 5 - Shake (Error State):**

```css
.shake {
  animation: shake 0.5s ease;
}

@keyframes shake {
  0%, 100% {
    transform: translateX(0);
  }
  10%, 30%, 50%, 70%, 90% {
    transform: translateX(-10px);
  }
  20%, 40%, 60%, 80% {
    transform: translateX(10px);
  }
}
```

## Performance Best Practices

Not all CSS properties are equal when it comes to animation performance.

### GPU-Accelerated Properties (Fast)

**Animate these for best performance:**
- `transform` (translate, rotate, scale)
- `opacity`

These properties don't trigger layout or paint - they're handled by the GPU.

```css
/* Good - GPU accelerated */
.element {
  transition: transform 0.3s, opacity 0.3s;
}

.element:hover {
  transform: translateY(-5px);
  opacity: 0.8;
}
```

### Expensive Properties (Slow)

**Avoid animating these:**
- `width`, `height`
- `margin`, `padding`
- `top`, `left`, `right`, `bottom`
- `border-width`

These force the browser to recalculate layout (expensive).

```css
/* Bad - forces layout recalculation */
.element {
  transition: width 0.3s, height 0.3s;
}

.element:hover {
  width: 300px;
  height: 200px;
}
```

### Use will-change for Heavy Animations

Tell the browser to optimize for animation:

```css
.element {
  will-change: transform, opacity;
}
```

**Important:** Only use `will-change` when actually animating. Remove it when done to free memory.

### Reduce Motion for Accessibility

Some users prefer reduced motion. Respect their preference:

```css
/* Normal animations */
.element {
  transition: transform 0.3s ease;
}

/* Disable for users who prefer reduced motion */
@media (prefers-reduced-motion: reduce) {
  .element {
    transition: none;
  }
}
```

## Common Animation Patterns

**Pattern 1: Hover Lift**

```css
.card {
  transition: transform 0.3s ease, box-shadow 0.3s ease;
}

.card:hover {
  transform: translateY(-8px);
  box-shadow: 0 12px 24px rgba(0,0,0,0.15);
}
```

**Pattern 2: Loading State**

```css
.button-loading::after {
  content: "";
  width: 16px;
  height: 16px;
  border: 2px solid white;
  border-top-color: transparent;
  border-radius: 50%;
  animation: spin 0.6s linear infinite;
}
```

**Pattern 3: Attention Seeker**

```css
@keyframes shake {
  0%, 100% { transform: translateX(0); }
  25% { transform: translateX(-5px); }
  75% { transform: translateX(5px); }
}

.error-input {
  animation: shake 0.3s ease;
  border-color: #ef4444;
}
```

## Best Practices

1. **Keep animations short** - 200-500ms for transitions, 1-2s for animations
2. **Use ease-in-out for natural motion** - Avoid linear unless intentional
3. **Animate transform and opacity** - Best performance
4. **Provide reduced motion option** - Use `prefers-reduced-motion`
5. **Don't overuse animations** - Too much motion is distracting
6. **Test on real devices** - Performance varies

## Common Mistakes to Avoid

1. **Animating expensive properties**
   - Avoid: width, height, margins
   - Use: transform, opacity

2. **Forgetting prefixes for old browsers**
   - Use autoprefixer or build tools

3. **Too many simultaneous animations**
   - Can cause jank on slower devices
   - Limit concurrent animations

4. **Not considering accessibility**
   - Respect prefers-reduced-motion
   - Don't rely solely on animation to convey information

## Key Takeaways

- **Transitions** smooth changes between states, need a trigger
- **Animations** run automatically, can loop, use @keyframes
- **transform** property includes translate, rotate, scale, skew
- **Animate transform and opacity** for best performance
- Use **ease-in-out** for natural-feeling motion
- **animation-iteration-count: infinite** creates looping animations
- **will-change** optimizes performance for heavy animations
- Respect **prefers-reduced-motion** for accessibility
- Keep animations short (200-500ms) and purposeful
- Test performance on real devices

CSS animations and transitions bring interfaces to life. Use them thoughtfully to guide users, provide feedback, and create delightful experiences.
