# JavaScript Fundamentals - Quick Reference

> **Quick Summary:** Core JavaScript syntax covering variables, data types, functions, arrays, objects, control flow, ES6+ features, string methods, and the this keyword.

## Variables & Scope

| Declaration | Scope | Reassignable | Hoisted | Use When |
|-------------|-------|--------------|---------|----------|
| `const` | Block | ❌ No | Yes (TDZ) | Default choice - prevents reassignment |
| `let` | Block | ✅ Yes | Yes (TDZ) | Need to reassign value |
| `var` | Function | ✅ Yes | Yes (undefined) | Never - legacy only |

**Temporal Dead Zone (TDZ):** Period from scope entry until `let`/`const` initialization. Accessing during TDZ throws `ReferenceError`.

**Key Rules:**
- Use `const` by default
- Use `let` only when reassignment needed
- Avoid `var` in modern code
- Block scope: `{ }` in if, for, while, functions

## Data Types

### Primitives (7 types)

| Type | Example | typeof | Notes |
|------|---------|--------|-------|
| `string` | `"hello"`, `'world'`, `` `template` `` | `"string"` | Immutable text |
| `number` | `42`, `3.14`, `NaN`, `Infinity` | `"number"` | Integers & decimals |
| `boolean` | `true`, `false` | `"boolean"` | Logical values |
| `null` | `null` | `"object"` | Intentional empty (bug!) |
| `undefined` | `undefined` | `"undefined"` | Not yet set |
| `symbol` | `Symbol('id')` | `"symbol"` | Unique identifiers |
| `bigint` | `123n` | `"bigint"` | Large integers |

### Reference Types

| Type | Example | typeof | Notes |
|------|---------|--------|-------|
| Object | `{ name: "Alice" }` | `"object"` | Key-value pairs |
| Array | `[1, 2, 3]` | `"object"` | Ordered list (special object) |
| Function | `function() {}` | `"function"` | Callable object |

### Primitive vs Reference

| Aspect | Primitive | Reference |
|--------|-----------|-----------|
| **Storage** | Value directly | Memory address |
| **Copying** | Creates new value | Shares same object |
| **Comparison** | By value | By reference |

```javascript
// Primitive - copy value
let a = 10;
let b = a;
b = 20;  // a is still 10

// Reference - share object
let obj1 = { x: 1 };
let obj2 = obj1;
obj2.x = 2;  // obj1.x is also 2!
```

### Type Coercion Quick Hits

| Expression | Result | Why |
|------------|--------|-----|
| `5 + "5"` | `"55"` | Number → string (concatenation) |
| `"10" - 5` | `5` | String → number |
| `5 == "5"` | `true` | Coerces types |
| `5 === "5"` | `false` | No coercion (use this!) |
| `Boolean(0)` | `false` | 0 is falsy |
| `Boolean([])` | `true` | Empty array is truthy |

**Falsy Values:** `false`, `0`, `""`, `null`, `undefined`, `NaN` (everything else is truthy)

## Functions

### Three Ways to Define

```javascript
// 1. Function Declaration (hoisted)
function add(a, b) {
  return a + b;
}

// 2. Function Expression (not hoisted)
const multiply = function(a, b) {
  return a * b;
};

// 3. Arrow Function (lexical this)
const divide = (a, b) => a / b;  // Implicit return
const subtract = (a, b) => {     // Explicit return
  return a - b;
};
```

### Arrow Function Shortcuts

| Syntax | Use Case |
|--------|----------|
| `() => value` | No parameters |
| `x => x * 2` | Single parameter (parentheses optional) |
| `(a, b) => a + b` | Implicit return (no braces) |
| `(a, b) => { return a + b; }` | Explicit return (with braces) |

### this Binding

| Function Type | this Binding |
|---------------|--------------|
| Regular function in object | Object that called it (implicit) |
| Standalone function | Global/undefined (default) |
| Arrow function | Inherited from outer scope (lexical) |

**Rule:** Use regular functions for object methods, arrow functions for callbacks.

## Arrays & Objects

### Essential Array Methods

| Method | Purpose | Returns | Mutates Original? |
|--------|---------|---------|-------------------|
| `map(fn)` | Transform each element | New array | ❌ No |
| `filter(fn)` | Select elements | New array | ❌ No |
| `find(fn)` | First match | Element or undefined | ❌ No |
| `reduce(fn, init)` | Combine to single value | Any type | ❌ No |
| `push(item)` | Add to end | New length | ✅ Yes |
| `pop()` | Remove from end | Removed item | ✅ Yes |
| `slice(start, end)` | Extract portion | New array | ❌ No |
| `includes(val)` | Check existence | Boolean | ❌ No |

```javascript
// Chaining methods
[1, 2, 3, 4, 5]
  .filter(n => n % 2 === 0)  // [2, 4]
  .map(n => n * 2)            // [4, 8]
  .reduce((sum, n) => sum + n, 0);  // 12
```

### Destructuring

```javascript
// Array Destructuring
const [first, second, ...rest] = [1, 2, 3, 4, 5];
// first = 1, second = 2, rest = [3, 4, 5]

// Object Destructuring
const { name, age = 18 } = { name: "Alice", city: "NYC" };
// name = "Alice", age = 18 (default), city not extracted

// Rename variables
const { name: userName } = { name: "Bob" };
// userName = "Bob"
```

### Spread Operator (...)

```javascript
// Copy array
const copy = [...original];

// Merge arrays
const merged = [...arr1, ...arr2];

// Copy object
const objCopy = { ...original };

// Merge objects (later properties overwrite)
const settings = { ...defaults, ...userPrefs };

// Add properties
const updated = { ...user, age: 31 };
```

## Control Flow

### Conditionals

```javascript
// if/else
if (condition) {
  // code
} else if (otherCondition) {
  // code
} else {
  // code
}

// Ternary operator (single expression)
const result = condition ? valueIfTrue : valueIfFalse;

// Logical operators
condition1 && condition2  // AND
condition1 || condition2  // OR
!condition                // NOT
```

### Loops

| Loop Type | Use Case | Syntax |
|-----------|----------|--------|
| `for` | Known iterations | `for (let i = 0; i < 10; i++)` |
| `for...of` | Array values | `for (const item of array)` |
| `while` | Unknown iterations | `while (condition)` |
| `do...while` | Run at least once | `do { } while (condition)` |

```javascript
// Array iteration best practices
for (const item of items) { }  // Values only
for (let i = 0; i < items.length; i++) { }  // Need index
items.forEach(item => { });  // Functional style
items.map(item => transform(item));  // Transform
```

### Loop Control

- `break` - Exit loop immediately
- `continue` - Skip to next iteration

## ES6+ Features

### Template Literals

```javascript
// String interpolation
const name = "Alice";
const age = 25;
const message = `Hello, I'm ${name} and I'm ${age} years old.`;

// Multi-line strings
const html = `
  <div>
    <h1>${title}</h1>
    <p>${content}</p>
  </div>
`;

// Expressions inside ${}
`Total: $${price * (1 + tax)}`
`Status: ${isActive ? "Active" : "Inactive"}`
```

### Default Parameters

```javascript
const greet = (name = "Guest", greeting = "Hello") => {
  return `${greeting}, ${name}!`;
};

greet("Alice", "Hi");  // "Hi, Alice!"
greet("Bob");          // "Hello, Bob!"
greet();               // "Hello, Guest!"
```

### Rest Operator (Function Parameters)

```javascript
// Collect remaining arguments into array
const sum = (...numbers) => {
  return numbers.reduce((total, n) => total + n, 0);
};

sum(1, 2, 3);       // 6
sum(5, 10, 15, 20); // 50

// Combined with named parameters
const log = (level, message, ...details) => {
  console.log(`[${level}] ${message}`, details);
};
```

### Rest vs Spread

| Operator | Context | Purpose | Example |
|----------|---------|---------|---------|
| **Rest** | Function parameters | Collect into array | `function(...args)` |
| **Spread** | Function calls, arrays, objects | Expand from array/object | `func(...arr)`, `[...arr]` |

## String Methods

### Extracting

| Method | Purpose | Example |
|--------|---------|---------|
| `slice(start, end)` | Extract portion | `"Hello".slice(1, 4)` → `"ell"` |
| `slice(-n)` | Last n characters | `"Hello".slice(-2)` → `"lo"` |

### Searching

| Method | Returns | Example |
|--------|---------|---------|
| `includes(str)` | Boolean | `"Hello".includes("ell")` → `true` |
| `indexOf(str)` | Index or -1 | `"Hello".indexOf("l")` → `2` |
| `startsWith(str)` | Boolean | `"Hello".startsWith("He")` → `true` |
| `endsWith(str)` | Boolean | `"file.pdf".endsWith(".pdf")` → `true` |

### Transforming

| Method | Purpose | Example |
|--------|---------|---------|
| `toUpperCase()` | Convert to uppercase | `"hello"` → `"HELLO"` |
| `toLowerCase()` | Convert to lowercase | `"HELLO"` → `"hello"` |
| `trim()` | Remove whitespace (both ends) | `"  hi  "` → `"hi"` |
| `replace(old, new)` | Replace first occurrence | `"hello".replace("l", "L")` → `"heLlo"` |
| `replaceAll(old, new)` | Replace all occurrences | `"hello".replaceAll("l", "L")` → `"heLLo"` |
| `split(sep)` | String → array | `"a,b,c".split(",")` → `["a","b","c"]` |
| `padStart(len, char)` | Pad beginning | `"5".padStart(3, "0")` → `"005"` |
| `padEnd(len, char)` | Pad end | `"5".padEnd(3, "0")` → `"500"` |
| `repeat(count)` | Repeat string | `"*".repeat(5)` → `"*****"` |

### Common Patterns

```javascript
// Capitalize first letter
const capitalize = str => str.charAt(0).toUpperCase() + str.slice(1);

// Title case
const titleCase = str => str.toLowerCase()
  .split(" ")
  .map(word => capitalize(word))
  .join(" ");

// Slugify
const slugify = str => str.toLowerCase()
  .trim()
  .replace(/[^\w\s-]/g, "")
  .replace(/\s+/g, "-");
```

## The this Keyword

### Binding Rules

| Call Style | this Value | Example |
|------------|------------|---------|
| `obj.method()` | `obj` (implicit) | Object before the dot |
| `function()` | Global or undefined | Standalone call |
| Arrow function | Inherited (lexical) | From outer scope |
| `new Constructor()` | New object | Constructor pattern |

### Common Pitfalls & Solutions

| Problem | Broken Code | Fixed Code |
|---------|-------------|------------|
| Lost binding | `const fn = obj.method; fn();` | `obj.method()` or use arrow function |
| Callbacks | `arr.forEach(function() { this })` | `arr.forEach(() => { this })` |
| Event handlers | `btn.addEventListener('click', obj.method)` | `btn.addEventListener('click', () => obj.method())` |
| Timers | `setTimeout(function() { this }, 1000)` | `setTimeout(() => { this }, 1000)` |

### When to Use What

**Regular functions:** Object methods, need dynamic `this`
**Arrow functions:** Callbacks, event handlers, timers (preserve outer `this`)

## Quick Syntax Comparison

### Old vs New

| Old Way | New Way (ES6+) |
|---------|----------------|
| `var x = 5;` | `const x = 5;` or `let x = 5;` |
| `"Hello " + name` | `` `Hello ${name}` `` |
| `function(a) { if (!a) a = 10; }` | `function(a = 10) { }` |
| `function.apply(null, arr)` | `function(...arr)` |
| `var x = arr[0], y = arr[1];` | `const [x, y] = arr;` |
| `var name = obj.name;` | `const { name } = obj;` |
| `arr1.concat(arr2)` | `[...arr1, ...arr2]` |

## Interview Quick Hits

| Question | Answer |
|----------|--------|
| **What's the difference between let, const, and var?** | `let` and `const` are block-scoped and have TDZ; `var` is function-scoped and hoisted as `undefined`. `const` prevents reassignment. |
| **What is hoisting?** | JavaScript moves declarations to the top of scope before execution. `var` is hoisted and initialized as `undefined`. `let`/`const` are hoisted but stay in TDZ until declaration. |
| **What are the 7 primitive types?** | `string`, `number`, `boolean`, `null`, `undefined`, `symbol`, `bigint` |
| **What's the difference between null and undefined?** | `null` is intentional absence (you set it). `undefined` means not yet assigned (JavaScript sets it). |
| **Why use === instead of ==?** | `===` checks type and value without coercion. `==` coerces types, causing unexpected results (`5 == "5"` is `true`). |
| **What does map() do?** | Creates a new array by transforming each element with a function. Doesn't modify original array. |
| **What is destructuring?** | Syntax to extract values from arrays/objects into variables: `const { name, age } = user;` |
| **What's the spread operator?** | `...` expands arrays/objects. Used for copying, merging, and passing elements individually. |
| **When does this refer to the global object?** | In standalone function calls (non-strict mode) or when a method loses its binding. |
| **Why use arrow functions?** | Concise syntax, implicit return for single expressions, and lexical `this` (inherit from outer scope). |
| **What's the Temporal Dead Zone?** | Period from scope entry to `let`/`const` initialization. Accessing variables during TDZ throws `ReferenceError`. |
| **What's type coercion?** | Automatic type conversion by JavaScript. `5 + "5"` becomes `"55"` (number → string). Use explicit conversion or `===`. |
| **What are falsy values?** | `false`, `0`, `""`, `null`, `undefined`, `NaN`. Everything else is truthy (including `[]` and `{}`). |
| **What does Object.freeze() do?** | Makes an object immutable (prevents adding/changing/deleting properties). `const` only prevents reassignment, not mutation. |
| **When do arrow functions fail?** | As object methods (need dynamic `this`), when you need `arguments` object, or as constructors (can't use `new`). |

## Common Gotchas

| Issue | Problem | Solution |
|-------|---------|----------|
| `typeof null` | Returns `"object"` | Check with `value === null` |
| `typeof []` | Returns `"object"` | Use `Array.isArray(value)` |
| Array copying | `const b = a` shares reference | Use `const b = [...a]` or `a.slice()` |
| Object copying | `const b = a` shares reference | Use `const b = { ...a }` |
| Lost `this` | Extract method and call standalone | Use arrow function or `.bind()` |
| `replace()` | Only replaces first occurrence | Use `replaceAll()` or regex with `/g` |
| Falsy checks | `0`, `""` treated as falsy | Check explicitly: `if (value !== null)` |
| Arrow function return | `x => { x * 2 }` returns `undefined` | Remove braces: `x => x * 2` or add `return` |

## Performance Tips

- Use `const` - helps JS engines optimize
- Avoid `delete` operator - makes objects slow
- Use array methods instead of manual loops
- Use `for...of` or `.forEach()` instead of traditional `for` loops for readability
- Cache array length in performance-critical loops: `for (let i = 0, len = arr.length; i < len; i++)`
- Use `===` to avoid coercion overhead
- Shallow copy with spread is fine; use libraries for deep copy
