## Essential Commands

| Command | Purpose |
|---------|---------|
| `git init` | Initialize new repository |
| `git clone url` | Copy remote repository |
| `git status` | Show working directory state |
| `git add file` | Stage changes |
| `git add .` | Stage all changes |
| `git commit -m "msg"` | Save staged changes |
| `git log --oneline` | View commit history |
| `git diff` | Show unstaged changes |

## Branching & Merging

| Command | Purpose |
|---------|---------|
| `git branch` | List branches |
| `git branch name` | Create branch |
| `git checkout name` | Switch branch |
| `git checkout -b name` | Create and switch |
| `git switch name` | Switch branch (modern) |
| `git switch -c name` | Create and switch (modern) |
| `git merge name` | Merge branch into current |
| `git branch -d name` | Delete merged branch |
| `git branch -D name` | Force delete branch |

### Branch Naming Conventions

| Pattern | Use Case |
|---------|----------|
| `feature/name` | New features |
| `bugfix/name` | Bug fixes |
| `hotfix/name` | Urgent production fixes |
| `release/version` | Release preparation |

## Remote Repositories

| Command | Purpose |
|---------|---------|
| `git remote -v` | List remotes with URLs |
| `git remote add name url` | Add remote |
| `git push origin branch` | Push to remote |
| `git push -u origin branch` | Push and set upstream |
| `git pull` | Fetch and merge |
| `git fetch` | Download without merging |
| `git push --tags` | Push all tags |

## Git Flow Branches

| Branch | Purpose | Merges To |
|--------|---------|-----------|
| `main` | Production code | - |
| `develop` | Integration | main (via release) |
| `feature/*` | New features | develop |
| `release/*` | Release prep | main + develop |
| `hotfix/*` | Urgent fixes | main + develop |

## Rebase vs Merge

| Aspect | Merge | Rebase |
|--------|-------|--------|
| History | Preserves branches | Linear |
| Commits | Adds merge commit | Rewrites hashes |
| Safety | Safe for shared branches | Local only |
| Undo | Easy | Harder |

### Golden Rule
> Never rebase commits that have been pushed and shared with others.

### Interactive Rebase Actions

| Action | Purpose |
|--------|---------|
| `pick` | Use commit as-is |
| `reword` | Edit commit message |
| `squash` | Combine with previous |
| `fixup` | Combine, discard message |
| `drop` | Remove commit |

```bash
git rebase -i HEAD~3  # Edit last 3 commits
```

## Cherry-Picking

```bash
# Copy specific commit to current branch
git cherry-pick abc123

# Include source in message
git cherry-pick -x abc123

# Stage only, don't commit
git cherry-pick -n abc123
```

## Stashing Changes

| Command | Purpose |
|---------|---------|
| `git stash` | Save changes |
| `git stash push -m "msg"` | Save with message |
| `git stash -u` | Include untracked files |
| `git stash list` | Show all stashes |
| `git stash show -p` | Show stash diff |
| `git stash pop` | Apply and remove |
| `git stash apply` | Apply and keep |
| `git stash drop` | Delete a stash |
| `git stash clear` | Delete all stashes |

## Resolving Conflicts

### Conflict Markers

```
<<<<<<< HEAD
Your current changes
=======
Incoming changes
>>>>>>> branch-name
```

### Resolution Steps

1. Edit file - choose what to keep
2. Remove all conflict markers
3. `git add filename`
4. `git commit` (or `git merge --continue`)

### Quick Commands

| Command | Purpose |
|---------|---------|
| `git merge --abort` | Cancel merge |
| `git rebase --abort` | Cancel rebase |
| `git checkout --ours file` | Keep your version |
| `git checkout --theirs file` | Keep their version |

## Git Hooks

| Hook | Runs | Common Use |
|------|------|------------|
| `pre-commit` | Before commit | Lint, format |
| `commit-msg` | After message entered | Validate format |
| `pre-push` | Before push | Run tests |
| `post-commit` | After commit | Notifications |

### Hook Location
```bash
.git/hooks/pre-commit  # Must be executable
```

### Skip Hooks
```bash
git commit --no-verify
git push --no-verify
```

## Tags & Releases

| Command | Purpose |
|---------|---------|
| `git tag` | List tags |
| `git tag v1.0.0` | Lightweight tag |
| `git tag -a v1.0.0 -m "msg"` | Annotated tag |
| `git tag -a v1.0.0 abc123` | Tag specific commit |
| `git show v1.0.0` | Show tag details |
| `git push origin v1.0.0` | Push single tag |
| `git tag -d v1.0.0` | Delete local tag |
| `git checkout v1.0.0` | Checkout tag (detached) |

### Semantic Versioning

| Change Type | Version Change | Example |
|-------------|----------------|---------|
| Breaking changes | MAJOR | v1.0.0 -> v2.0.0 |
| New features | MINOR | v1.0.0 -> v1.1.0 |
| Bug fixes | PATCH | v1.0.0 -> v1.0.1 |

## Quick Workflows

### Feature Branch

```bash
git checkout develop
git checkout -b feature/name
# ... work and commit ...
git checkout develop
git merge feature/name
git branch -d feature/name
```

### Hotfix

```bash
git checkout main
git checkout -b hotfix/name
# ... fix and commit ...
git checkout main && git merge hotfix/name
git checkout develop && git merge hotfix/name
git tag -a v1.0.1 -m "Hotfix"
```

### Update Branch Before PR

```bash
git fetch origin
git rebase origin/main
git push --force-with-lease
```

## Interview Quick Hits

| Question | Answer |
|----------|--------|
| What is Git? | A distributed version control system that tracks changes and enables collaboration |
| What's the difference between `git fetch` and `git pull`? | Fetch downloads changes without merging; pull fetches and merges in one step |
| When should you use rebase vs merge? | Rebase for clean linear history on local branches; merge for shared branches to preserve history |
| What is a Git hook? | Scripts that run automatically at certain Git events like pre-commit or pre-push |
| How do you undo the last commit? | `git reset --soft HEAD~1` keeps changes staged; `git reset --hard HEAD~1` discards changes |
| What is cherry-picking? | Copying a specific commit from one branch to another without merging the entire branch |

## Common Mistakes to Avoid

| Mistake | Solution |
|---------|----------|
| Committing secrets | Use `.gitignore`, never commit credentials |
| Working on main | Always create a branch |
| Vague commit messages | Explain why, not what |
| Force push shared branches | Use `--force-with-lease` only on your branches |
| Long-lived branches | Merge frequently |
| Forgetting to pull | Always `git pull` before starting work |
| Rebasing shared commits | Only rebase local, unpushed commits |
