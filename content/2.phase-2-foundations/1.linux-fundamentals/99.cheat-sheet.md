# Linux Fundamentals Cheat Sheet

## File System Hierarchy

| Directory | Purpose |
|-----------|---------|
| `/` | Root of filesystem |
| `/home` | User home directories |
| `/etc` | Configuration files |
| `/var` | Variable data (logs, caches) |
| `/var/log` | System and application logs |
| `/usr` | User programs |
| `/bin`, `/sbin` | Essential commands |
| `/tmp` | Temporary files (cleared on reboot) |
| `/opt` | Third-party software |
| `/root` | Root user's home |

## Navigation Commands

| Command | Purpose |
|---------|---------|
| `pwd` | Print current directory |
| `ls` | List directory contents |
| `ls -la` | List all files with details |
| `ls -lh` | Human-readable file sizes |
| `cd /path` | Change to absolute path |
| `cd ~` or `cd` | Go to home directory |
| `cd ..` | Go up one directory |
| `cd -` | Go to previous directory |

## File Operations

| Command | Purpose |
|---------|---------|
| `mkdir -p dir/sub` | Create nested directories |
| `touch file.txt` | Create empty file |
| `cp -r src/ dest/` | Copy directory recursively |
| `cp -i file dest/` | Copy with overwrite prompt |
| `mv old new` | Move or rename |
| `rm file` | Delete file |
| `rm -rf dir/` | Delete directory (caution!) |
| `cat file` | Display file contents |
| `head -n 20 file` | First 20 lines |
| `tail -n 50 file` | Last 50 lines |
| `tail -f file` | Follow file in real-time |
| `less file` | Scrollable viewer |

## File Permissions

### Permission Values
| Value | Permission |
|-------|------------|
| 4 | Read (r) |
| 2 | Write (w) |
| 1 | Execute (x) |

### Common Permission Settings
| Numeric | Symbolic | Use Case |
|---------|----------|----------|
| 755 | rwxr-xr-x | Scripts, executables |
| 644 | rw-r--r-- | Regular files |
| 600 | rw------- | Private files, SSH keys |
| 700 | rwx------ | Private directories |

### chmod Commands
```bash
chmod 755 script.sh      # Numeric notation
chmod +x script.sh       # Add execute for all
chmod u+x file           # Add execute for owner
chmod g-w file           # Remove write from group
chmod o=r file           # Set read-only for others
```

### Ownership
```bash
chown user file          # Change owner
chown user:group file    # Change owner and group
chown -R user:group dir/ # Recursive change
chgrp group file         # Change group only
```

## Users & Groups

| Command | Purpose |
|---------|---------|
| `whoami` | Current username |
| `id` | User ID and groups |
| `groups` | List your groups |
| `useradd -m user` | Create user with home dir |
| `passwd user` | Set user password |
| `usermod -aG group user` | Add user to group |
| `userdel -r user` | Delete user and home dir |
| `groupadd group` | Create new group |
| `sudo command` | Run as root |
| `sudo -i` | Open root shell |

**Key Files:**
- `/etc/passwd` - User accounts
- `/etc/shadow` - Encrypted passwords
- `/etc/group` - Group definitions
- `/etc/sudoers` - Sudo permissions (edit with `visudo`)

## Process Management

### Viewing Processes
| Command | Purpose |
|---------|---------|
| `ps aux` | All processes with details |
| `ps -ef --forest` | Process tree |
| `top` | Real-time process viewer |
| `htop` | Interactive process viewer |
| `pstree` | Show process hierarchy |

### Controlling Processes
| Command | Purpose |
|---------|---------|
| `kill PID` | Graceful termination (SIGTERM) |
| `kill -9 PID` | Force kill (SIGKILL) |
| `kill -HUP PID` | Reload configuration |
| `killall name` | Kill processes by name |
| `pkill pattern` | Kill matching pattern |

### Background Jobs
```bash
command &              # Run in background
jobs                   # List background jobs
fg %1                  # Bring job 1 to foreground
bg %1                  # Resume job 1 in background
Ctrl+Z                 # Suspend current process
nohup command &        # Run immune to hangups
```

## Cron Scheduling

### Crontab Syntax
```
* * * * * command
│ │ │ │ │
│ │ │ │ └── Day of week (0-7, Sun=0 or 7)
│ │ │ └──── Month (1-12)
│ │ └────── Day of month (1-31)
│ └──────── Hour (0-23)
└────────── Minute (0-59)
```

### Common Patterns
| Expression | Meaning |
|------------|---------|
| `* * * * *` | Every minute |
| `*/15 * * * *` | Every 15 minutes |
| `0 * * * *` | Every hour |
| `0 3 * * *` | Daily at 3:00 AM |
| `0 9 * * 1` | Mondays at 9:00 AM |
| `0 0 1 * *` | First of month |

### Crontab Commands
```bash
crontab -l             # List your cron jobs
crontab -e             # Edit your cron jobs
crontab -r             # Remove all cron jobs
```

**System directories:**
- `/etc/cron.daily/` - Run once per day
- `/etc/cron.hourly/` - Run every hour
- `/etc/cron.weekly/` - Run weekly
- `/etc/cron.d/` - Custom cron files

## Log Files & Journalctl

### Important Log Files
| File | Contents |
|------|----------|
| `/var/log/syslog` | System messages (Debian/Ubuntu) |
| `/var/log/messages` | System messages (RHEL) |
| `/var/log/auth.log` | Authentication (Debian/Ubuntu) |
| `/var/log/secure` | Authentication (RHEL) |

### journalctl Commands
| Command | Purpose |
|---------|---------|
| `journalctl -u svc` | Logs for specific service |
| `journalctl -f` | Follow logs in real-time |
| `journalctl -n 100` | Last 100 lines |
| `journalctl -p err` | Errors only |
| `journalctl -b` | Current boot only |
| `journalctl --since "1 hour ago"` | Time-filtered |

### Kernel Messages
```bash
dmesg                  # View kernel messages
dmesg -T               # Human-readable timestamps
dmesg --level=err      # Show only errors
```

## Quick Troubleshooting

**Service not starting:**
```bash
systemctl status svc
journalctl -u svc -n 50
```

**Permission denied:**
```bash
ls -la file
stat file
```

**Disk full:**
```bash
df -h
du -sh /var/log
```

**Process using high CPU:**
```bash
top
ps aux --sort=-%cpu | head
```

**Find failed logins:**
```bash
journalctl -u sshd | grep "Failed password"
```

## Interview Quick Hits

| Question | Answer |
|----------|--------|
| What is the difference between `/etc` and `/var`? | `/etc` stores configuration files; `/var` stores variable data like logs and caches |
| How do Linux file permissions work? | Three groups (owner, group, others) with read (4), write (2), execute (1) permissions that sum together |
| What does `chmod 755` mean? | Owner can read/write/execute (7), group and others can read/execute (5) |
| What's the difference between `kill` and `kill -9`? | `kill` sends SIGTERM for graceful shutdown; `kill -9` sends SIGKILL for forced termination |
| How do you make a service start at boot? | `systemctl enable servicename` |
| What does the cron expression `0 3 * * *` mean? | Run daily at 3:00 AM |
