# Types of Cloud Computing (IaaS, PaaS, SaaS)

Cloud services come in three models that differ in **how much the provider manages for you**. The more the provider manages, the less control you have -- but also the less work you do.

### IaaS -- Infrastructure as a Service

IaaS gives you **virtual hardware**: servers, storage, and networking. You manage everything from the operating system up. The provider manages the physical data center, hardware, and virtualization layer.

**You manage:** OS, middleware, runtime, applications, data
**Provider manages:** Physical servers, storage, networking, virtualization

**AWS examples:** Amazon EC2, Amazon S3, Amazon VPC

**Use when:** You need full control over your environment, want to run custom software, or need to migrate existing on-premises applications with minimal changes.

### PaaS -- Platform as a Service

PaaS gives you a **ready-to-use platform** to deploy your code. The provider manages the OS, runtime, and middleware. You just upload your application.

**You manage:** Application code, data
**Provider manages:** OS, runtime, middleware, servers, storage, networking

**AWS examples:** AWS Elastic Beanstalk, AWS Lambda, Amazon RDS

**Use when:** You want to focus on writing code without managing servers, patching operating systems, or configuring web servers.

### SaaS -- Software as a Service

SaaS gives you a **complete application** accessed through a browser or API. The provider manages everything. You just use the software.

**You manage:** Your data and user settings
**Provider manages:** Everything else

**Examples:** Gmail, Salesforce, Microsoft 365, Slack, Zoom

**Use when:** You need a ready-made tool and do not want to build, deploy, or maintain the software yourself.

## Responsibility Comparison

This table shows who manages what in each model:

| Layer | On-Premises | IaaS | PaaS | SaaS |
|-------|------------|------|------|------|
| **Applications** | You | You | You | Provider |
| **Data** | You | You | You | You |
| **Runtime** | You | You | Provider | Provider |
| **Middleware** | You | You | Provider | Provider |
| **OS** | You | You | Provider | Provider |
| **Virtualization** | You | Provider | Provider | Provider |
| **Servers** | You | Provider | Provider | Provider |
| **Storage** | You | Provider | Provider | Provider |
| **Networking** | You | Provider | Provider | Provider |

The key pattern: as you move from IaaS to SaaS, the provider takes on more responsibility and you take on less.

## Choosing the Right Model

| Question | IaaS | PaaS | SaaS |
|----------|------|------|------|
| Need full OS control? | Yes | No | No |
| Want to deploy custom code? | Yes | Yes | No |
| Just need a working app? | No | No | Yes |
| Team has infrastructure skills? | Required | Helpful | Not needed |
| Speed to deploy? | Hours | Minutes | Instant |

## Key Takeaways

- **IaaS** gives you virtual hardware with full control (EC2, S3)
- **PaaS** gives you a platform to deploy code without managing servers (Elastic Beanstalk)
- **SaaS** gives you complete software ready to use (Gmail, Salesforce)
- Moving from IaaS to SaaS trades control for convenience
- Most organizations use a mix of all three models
