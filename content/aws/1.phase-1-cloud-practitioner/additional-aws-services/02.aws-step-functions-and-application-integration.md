# AWS Step Functions & Application Integration

**AWS Step Functions** lets you coordinate multiple AWS services into serverless workflows. Instead of writing complex code to manage retries, error handling, and sequencing, you define a visual workflow called a **state machine** and Step Functions runs it for you.

## How Step Functions Works

A Step Functions workflow is a **state machine** -- a series of steps where each step is a **state**. Each state does something: call a Lambda function, write to a database, wait for approval, or make a decision.

You define workflows using **Amazon States Language (ASL)**, a JSON-based language. Step Functions also provides a visual **Workflow Studio** where you drag and drop states to build workflows without writing JSON.

## State Types

Step Functions provides several state types for different tasks:

| State Type | What It Does | Example |
|------------|-------------|---------|
| **Task** | Performs work by calling an AWS service or API | Invoke a Lambda function |
| **Choice** | Adds branching logic (if/else) | Route orders over $100 to a review step |
| **Parallel** | Runs multiple branches at the same time | Check inventory AND validate address simultaneously |
| **Wait** | Pauses for a set time or until a timestamp | Wait 24 hours before sending a reminder |
| **Map** | Iterates over a list and runs a step for each item | Process each line item in an order |
| **Pass** | Passes input to output (useful for data transformation) | Reshape data between steps |
| **Succeed / Fail** | Marks the workflow as complete or failed | End the workflow with a result |

## Workflow Types

Step Functions offers two workflow types:

| Feature | Standard | Express |
|---------|----------|---------|
| **Max duration** | 1 year | 5 minutes |
| **Execution rate** | 2,000 per second | 100,000 per second |
| **Delivery** | Exactly once | At least once |
| **Pricing** | Per state transition | Per execution + duration |
| **Best for** | Order processing, approvals, ETL pipelines | IoT data processing, streaming transforms |

**Choose Standard** when your workflow runs for minutes, hours, or days and you need exactly-once execution. **Choose Express** when you need high throughput for short, fast tasks.

## Built-In Error Handling

Step Functions handles errors automatically with two mechanisms:

- **Retry** -- re-run a failed state with configurable intervals and max attempts
- **Catch** -- redirect to a fallback state when retries are exhausted

This removes the need to write error-handling code inside each service.

## Service Integrations

Step Functions connects directly to over 200 AWS services:

- **AWS Lambda** -- run custom code
- **Amazon DynamoDB** -- read and write data
- **Amazon SQS** -- send messages
- **Amazon SNS** -- send notifications
- **Amazon ECS/Fargate** -- run containers
- **AWS Glue** -- run ETL jobs

## Other Integration Services

AWS provides additional services for connecting applications:

| Service | Purpose | When to Use |
|---------|---------|-------------|
| **Amazon MQ** | Managed message broker (ActiveMQ, RabbitMQ) | Migrating existing messaging apps to AWS |
| **Amazon AppFlow** | Automated data flow between SaaS apps and AWS | Sync Salesforce data to S3 |
| **AWS AppSync** | Managed GraphQL and Pub/Sub API service | Real-time data sync for mobile and web apps |

## Key Takeaways

- Step Functions coordinates multi-step workflows as visual state machines
- **Standard workflows** run up to 1 year with exactly-once execution; **Express workflows** handle high-volume, short tasks
- State types include Task, Choice, Parallel, Wait, Map, Pass, Succeed, and Fail
- Built-in Retry and Catch handle errors without custom code
- Step Functions integrates directly with over 200 AWS services
- Other integration services like Amazon MQ and AppFlow connect legacy and SaaS applications to AWS
