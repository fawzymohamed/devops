# Amazon CloudWatch Monitoring

**Amazon CloudWatch** monitors your AWS resources and applications in real time. It collects performance data (metrics), triggers alerts (alarms), and provides visual overviews (dashboards) so you can spot problems before your users do.

## Core Concepts

### Metrics

A **metric** is a time-ordered set of data points that tracks a specific measurement. AWS services automatically send metrics to CloudWatch.

Common examples:

| AWS Service | Metric | What It Measures |
|-------------|--------|-----------------|
| **EC2** | `CPUUtilization` | Percentage of CPU in use |
| **EC2** | `NetworkIn` / `NetworkOut` | Bytes of network traffic |
| **RDS** | `DatabaseConnections` | Number of active database connections |
| **S3** | `BucketSizeBytes` | Total storage used |
| **Lambda** | `Invocations` | Number of function calls |
| **Lambda** | `Duration` | How long each invocation takes |
| **ELB** | `RequestCount` | Number of requests received |

**Basic monitoring** sends metrics every **5 minutes** at no extra cost. **Detailed monitoring** sends metrics every **1 minute** for an additional fee.

### Custom Metrics

AWS services report built-in metrics, but you can also publish **custom metrics** from your own applications. For example, you could track:

- Number of user sign-ups per minute
- Order processing time
- Items in a shopping cart

You send custom metrics using the AWS SDK or the CloudWatch Agent installed on your servers.

### Namespaces and Dimensions

- **Namespace** -- a container that groups related metrics (e.g., `AWS/EC2` for all EC2 metrics)
- **Dimension** -- a name/value pair that filters metrics (e.g., `InstanceId = i-0abc123` to see one specific instance)

## Alarms

A CloudWatch **alarm** watches a single metric and takes action when the value crosses a threshold you define.

**Three alarm states:**

| State | Meaning |
|-------|---------|
| **OK** | The metric is within the threshold |
| **ALARM** | The metric has breached the threshold |
| **INSUFFICIENT_DATA** | Not enough data to evaluate |

**Example:** Create an alarm that triggers when EC2 CPU utilization exceeds 80% for 5 consecutive minutes.

### Alarm Actions

When an alarm enters the ALARM state, it can:

- **Send a notification** via Amazon SNS (email, SMS, or trigger a Lambda function)
- **Trigger Auto Scaling** to add or remove EC2 instances
- **Perform EC2 actions** -- stop, terminate, reboot, or recover an instance

This lets you automate responses to performance issues without manual intervention.

## Dashboards

A CloudWatch **dashboard** is a customizable page that displays metrics as graphs, numbers, and charts. You can put metrics from different services on a single dashboard for a complete view of your application.

Dashboards support:

- Line graphs, stacked area charts, and number widgets
- Metrics from multiple Regions and accounts on one page
- Automatic refresh to show real-time data
- Sharing with team members

AWS also provides **automatic dashboards** for many services that show key metrics without any setup.

## CloudWatch Logs

**CloudWatch Logs** collects and stores log data from your AWS services and applications.

| Concept | What It Is |
|---------|-----------|
| **Log group** | A collection of log streams that share the same settings (e.g., all logs from one Lambda function) |
| **Log stream** | A sequence of log events from a single source (e.g., one EC2 instance) |
| **Log event** | A single log entry with a timestamp and message |

### What You Can Do with Logs

- **Search and filter** -- find specific error messages across thousands of log entries
- **Logs Insights** -- run queries to analyze log data and find patterns
- **Metric filters** -- turn log patterns into CloudWatch metrics (e.g., count how many times "ERROR" appears per minute)
- **Retention** -- store logs from 1 day to 10 years, or indefinitely

## Key Takeaways

- CloudWatch monitors AWS resources by collecting **metrics** -- time-series data points about performance
- **Basic monitoring** reports every 5 minutes (free); **detailed monitoring** reports every 1 minute (paid)
- **Alarms** watch metrics and trigger actions (SNS notifications, Auto Scaling, EC2 actions) when thresholds are breached
- **Dashboards** visualize metrics from multiple services in one place
- **CloudWatch Logs** stores and lets you search log data from AWS services and your applications
- **Custom metrics** let you track application-specific data alongside AWS metrics
