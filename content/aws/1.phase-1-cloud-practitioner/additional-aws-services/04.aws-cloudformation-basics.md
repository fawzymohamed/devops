# AWS CloudFormation Basics

**AWS CloudFormation** lets you define your entire AWS infrastructure in a text file called a **template**. You submit the template, and CloudFormation creates all the resources for you -- VPCs, EC2 instances, databases, and more. This approach is called **Infrastructure as Code (IaC)**.

## Why Infrastructure as Code?

Without IaC, you create resources manually through the AWS Console -- clicking through screens, typing values, and hoping you remember every setting. This causes problems:

| Manual Approach | Infrastructure as Code |
|----------------|----------------------|
| Hard to repeat exactly | Run the same template every time |
| No record of what changed | Track changes in version control (Git) |
| Environments drift apart | Identical dev, staging, and production |
| Cleanup is error-prone | Delete the stack and everything goes away |

IaC treats infrastructure like software: write it, version it, review it, and deploy it consistently.

## Key Concepts

### Templates

A **template** is a text file (YAML or JSON) that describes every AWS resource you want to create. YAML is more common because it is easier to read.

A basic template has these sections:

```yaml
AWSTemplateFormatVersion: "2010-09-09"
Description: "My first CloudFormation template"

Resources:
  MyS3Bucket:
    Type: AWS::S3::Bucket
    Properties:
      BucketName: my-example-bucket

  MyEC2Instance:
    Type: AWS::EC2::Instance
    Properties:
      InstanceType: t3.micro
      ImageId: ami-0abcdef1234567890
```

| Section | Purpose |
|---------|---------|
| **AWSTemplateFormatVersion** | Template format (always "2010-09-09") |
| **Description** | What this template does |
| **Resources** | The AWS resources to create (required) |
| **Parameters** | Input values you pass at stack creation time |
| **Outputs** | Values to display after the stack is created (e.g., a URL) |
| **Mappings** | Static lookup tables (e.g., AMI IDs per Region) |

**Resources** is the only required section. Every resource needs a **Type** (like `AWS::S3::Bucket`) and **Properties** that configure it.

### Stacks

A **stack** is a collection of AWS resources created from a template. CloudFormation manages all resources in a stack as a single unit.

- **Create a stack** -- CloudFormation reads your template and provisions all resources
- **Update a stack** -- modify the template and apply changes; CloudFormation figures out what to add, change, or remove
- **Delete a stack** -- CloudFormation removes all resources the stack created

If a resource fails to create, CloudFormation **rolls back** the entire stack to its previous state. This all-or-nothing behavior prevents partial deployments.

### Change Sets

A **change set** lets you preview what CloudFormation will do before it does it. When you update a stack:

1. Create a change set from your updated template
2. Review the proposed changes (which resources will be added, modified, or deleted)
3. Execute the change set to apply the changes, or cancel it

This prevents accidental changes to production resources.

CloudFormation handles **dependency ordering** automatically. If your EC2 instance needs a security group, CloudFormation creates the security group first, then the instance.

## Benefits

- **Repeatable** -- deploy the same infrastructure in any Region or account with one template
- **Version controlled** -- store templates in Git to track who changed what and when
- **Self-documenting** -- the template IS the documentation of your infrastructure
- **Safe updates** -- change sets let you preview before applying; rollback protects against failures
- **Free to use** -- you only pay for the AWS resources CloudFormation creates, not for CloudFormation itself

## Key Takeaways

- CloudFormation is AWS's Infrastructure as Code service -- define resources in templates, deploy them as stacks
- Templates are written in YAML or JSON; the **Resources** section is the only required part
- A **stack** manages all its resources as one unit: create, update, or delete them together
- **Change sets** let you preview updates before applying them
- Failed deployments roll back automatically to prevent broken infrastructure
- CloudFormation is free -- you only pay for the resources it creates
