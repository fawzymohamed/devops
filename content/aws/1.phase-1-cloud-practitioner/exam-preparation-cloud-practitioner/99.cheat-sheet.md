# Exam Preparation: Cloud Practitioner - Quick Reference

## CLF-C02 Exam Format

| Detail | Value |
|--------|-------|
| **Questions** | 65 total (50 scored + 15 unscored) |
| **Time** | 90 minutes (~1 min 23 sec per question) |
| **Passing score** | 700 / 1000 (scaled) |
| **Question types** | Multiple choice (pick 1) and multiple response (pick 2-3) |
| **Cost** | $100 USD |
| **Delivery** | Pearson VUE testing center or online proctored |
| **Penalty for guessing** | None -- always answer every question |

---

## Domain Weights

| Domain | Weight | Approx. Scored Questions | Focus Areas |
|--------|--------|--------------------------|-------------|
| **Cloud Concepts** | 24% | ~12 | Cloud benefits, IaaS/PaaS/SaaS, deployment models, Well-Architected Framework, cloud economics |
| **Security & Compliance** | 30% | ~15 | Shared Responsibility Model, IAM, encryption, compliance programs, security services |
| **Cloud Technology & Services** | 34% | ~17 | EC2, S3, Lambda, VPC, RDS, DynamoDB, CloudFront, Route 53, ELB, Auto Scaling |
| **Billing, Pricing & Support** | 12% | ~6 | Pricing models, Free Tier, Cost Explorer, Budgets, support plans, Trusted Advisor |

**Priority:** Technology & Services + Security = **64% of the exam**. Master these two first.

## Key AWS Services by Category

### Compute

| Service | One-Liner |
|---------|-----------|
| **EC2** | Virtual servers you manage (IaaS) |
| **Lambda** | Serverless functions -- pay per request and compute time |
| **ECS / Fargate** | Run containers; Fargate = serverless containers |
| **Elastic Beanstalk** | Deploy apps without managing infrastructure (PaaS) |
| **Lightsail** | Simple virtual servers for small workloads |

### Storage

| Service | One-Liner |
|---------|-----------|
| **S3** | Object storage for files, images, backups (11 nines durability) |
| **EBS** | Block storage attached to EC2 instances |
| **EFS** | Shared file storage across multiple EC2 instances |
| **S3 Glacier** | Archive storage with minutes-to-hours retrieval |

### Database

| Service | One-Liner |
|---------|-----------|
| **RDS** | Managed relational databases (MySQL, PostgreSQL, etc.) |
| **DynamoDB** | Managed NoSQL key-value database, single-digit ms latency |
| **Aurora** | AWS-built relational DB, MySQL/PostgreSQL compatible, 5x faster |
| **ElastiCache** | In-memory caching (Redis, Memcached) |
| **Redshift** | Data warehouse for analytics |

### Networking

| Service | One-Liner |
|---------|-----------|
| **VPC** | Isolated virtual network you control |
| **Route 53** | DNS service (domain name to IP) |
| **CloudFront** | CDN -- caches content at edge locations worldwide |
| **ELB** | Distributes traffic across instances in multiple AZs |
| **Direct Connect** | Dedicated private network connection to AWS |

### Security

| Service | One-Liner |
|---------|-----------|
| **IAM** | Users, groups, roles, and policies for access control |
| **GuardDuty** | Threat detection with machine learning |
| **WAF** | Blocks web exploits (SQL injection, XSS) |
| **Shield** | DDoS protection (Standard = free, Advanced = paid) |
| **Inspector** | Scans for software vulnerabilities |
| **Macie** | Discovers sensitive data in S3 |
| **Artifact** | Download compliance reports (SOC, ISO, PCI) |
| **CloudTrail** | Logs all API calls for audit |

### Cost Management

| Tool | Purpose |
|------|---------|
| **Pricing Calculator** | Estimate costs **before** deployment |
| **Cost Explorer** | Visualize actual spending **after** deployment |
| **Budgets** | Set spending alerts and thresholds |
| **Trusted Advisor** | Best practice checks (cost, security, performance, fault tolerance, service limits) |

## Critical Concepts to Remember

### Shared Responsibility Model

| AWS Manages (Security OF the Cloud) | Customer Manages (Security IN the Cloud) |
|--------------------------------------|------------------------------------------|
| Physical data centers | IAM users, groups, roles, policies |
| Hypervisor and host OS | Guest OS patching on EC2 |
| Network infrastructure | Security group and NACL configuration |
| Global infrastructure | Data encryption choices |
| Managed service patching (RDS, Lambda) | Application code and firewall rules |

**Rule of thumb:** If you configure it in the console, it is your responsibility.

### Well-Architected Framework -- 6 Pillars

| Pillar | Core Focus |
|--------|------------|
| **Operational Excellence** | Run and improve systems as code |
| **Security** | Protect data, least privilege, encrypt |
| **Reliability** | Recover from failures, scale horizontally |
| **Performance Efficiency** | Use the right resource types, go serverless |
| **Cost Optimization** | Pay only for what you use, analyze spending |
| **Sustainability** | Minimize environmental impact |

### Pricing Models

| Model | Best For | Savings |
|-------|----------|---------|
| **On-Demand** | Unpredictable, short-term, dev/test | 0% (baseline) |
| **Reserved / Savings Plans** | Steady-state, 24/7 production (1-3 year) | Up to 72% |
| **Spot** | Fault-tolerant, interruptible batch jobs | Up to 90% |

### Support Plans

| Feature | Basic | Developer | Business | Enterprise |
|---------|-------|-----------|----------|------------|
| **Price** | Free | $29+/mo | $100+/mo | $15,000+/mo |
| **Technical support** | None | Email only | 24/7 phone/chat | 24/7 phone/chat |
| **Trusted Advisor** | Core checks | Core checks | Full (400+) | Full (400+) |
| **TAM** | No | No | No | Yes |
| **Prod-down response** | N/A | N/A | < 1 hour | 15 minutes |

## Exam Strategy

### Two-Pass Approach

1. **First pass (60 min):** Answer every question. Flag uncertain ones. Do not spend more than 2 minutes on any question.
2. **Second pass (20 min):** Return to flagged questions with fresh eyes.
3. **Final check (10 min):** Verify all questions are answered.

### Elimination Technique

- Read the question twice -- watch for MOST, LEAST, NOT, BEST
- Eliminate obviously wrong options first
- If two answers seem correct, pick the more complete one
- Never leave a question blank

### Study Timeline

| Time Remaining | Focus |
|---------------|-------|
| **4 weeks** | Cloud Concepts + Security fundamentals |
| **3 weeks** | Core services (EC2, S3, RDS, Lambda, VPC) |
| **2 weeks** | Additional services + Billing & Support |
| **1 week** | Practice exams + review weak areas |
| **Day before** | Light review, rest well, no cramming |

## Common Exam Traps

| Trap | How to Avoid |
|------|-------------|
| Confusing S3 (object storage) with EBS (block storage) | S3 = files/backups; EBS = disk attached to EC2 |
| Confusing RDS (SQL) with DynamoDB (NoSQL) | RDS = relational; DynamoDB = key-value |
| Confusing Security Groups with NACLs | SGs = stateful, instance-level; NACLs = stateless, subnet-level |
| Confusing CloudTrail (API logs) with CloudWatch (metrics) | Trail = who did what; Watch = how things are performing |
| "Eliminate all responsibility" distractors | Cloud shifts responsibility -- it never eliminates it |
| Overthinking simple questions | CLF-C02 tests breadth, not depth |
| Spending too long on one question | Flag it, guess, move on -- come back later |

## Interview Quick Hits

| Question | Answer |
|----------|--------|
| What is the CLF-C02 exam? | AWS Certified Cloud Practitioner -- a foundational certification covering cloud concepts, security, core services, and billing. 65 questions in 90 minutes, 700/1000 to pass |
| What are the four CLF-C02 domains? | Cloud Concepts (24%), Security & Compliance (30%), Cloud Technology & Services (34%), Billing, Pricing & Support (12%) |
| What is the Shared Responsibility Model? | AWS secures the cloud infrastructure (hardware, network, facilities). Customers secure what they put in the cloud (data, IAM, OS patches, encryption) |
| Name the 6 Well-Architected pillars. | Operational Excellence, Security, Reliability, Performance Efficiency, Cost Optimization, Sustainability |
| When should you use Spot Instances? | For fault-tolerant, interruptible workloads like batch processing. Spot offers up to 90% savings but AWS can reclaim instances with 2-minute notice |
| Which support plan includes a TAM? | Enterprise Support ($15,000+/mo). Enterprise On-Ramp ($5,500/mo) includes a pool of TAMs |
| What is the difference between Cost Explorer and Budgets? | Cost Explorer visualizes past and forecasted spending. Budgets sets spending thresholds and sends alerts when costs approach or exceed them |
| What question types appear on the CLF-C02? | Multiple choice (one correct from four) and multiple response (two or more correct from five+). No fill-in-the-blank, drag-and-drop, or free-text |
