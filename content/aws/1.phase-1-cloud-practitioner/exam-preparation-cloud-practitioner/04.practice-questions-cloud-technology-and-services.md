# Practice Questions: Cloud Technology & Services

This domain covers **34% of the CLF-C02 exam** -- the heaviest domain. Questions test your knowledge of core AWS services across compute, storage, databases, networking, and more. Below are 10 realistic practice questions.

## Tips for Technology Questions

- **"Which service?"** questions are the most common -- know what each service does in one sentence
- **Similar services** appear as distractors -- know the difference between S3 vs. EBS, RDS vs. DynamoDB, EC2 vs. Lambda
- **Scenario-based questions** describe a workload and ask you to pick the best service

---

## Practice Questions

### Question 1

**A company needs to store large amounts of unstructured data like images, videos, and backups. Which AWS service is the BEST fit?**

- A) Amazon EBS
- B) Amazon S3
- C) Amazon RDS
- D) Amazon EFS

> **Answer: B** -- Amazon S3 is object storage designed for unstructured data at any scale. It is ideal for images, videos, backups, and static website hosting. EBS provides block storage attached to EC2 instances. RDS is for relational databases. EFS is shared file storage.

---

### Question 2

**Which AWS service translates domain names (like example.com) into IP addresses?**

- A) Amazon VPC
- B) Amazon CloudFront
- C) Amazon Route 53
- D) Elastic Load Balancing

> **Answer: C** -- Amazon Route 53 is the AWS DNS (Domain Name System) service. It translates human-readable domain names into IP addresses so browsers can find servers. It also supports health checks and routing policies for high availability.

---

### Question 3

**A developer wants to run a containerized application on AWS without managing the underlying servers. Which service should they use?**

- A) Amazon EC2
- B) Amazon ECS with AWS Fargate
- C) AWS Lambda
- D) Amazon Lightsail

> **Answer: B** -- Amazon ECS (Elastic Container Service) with AWS Fargate runs containers without managing EC2 instances. Fargate handles server provisioning and scaling. EC2 requires managing servers manually. Lambda runs functions, not containers. Lightsail is a simplified compute service.

---

### Question 4

**What is the purpose of an Elastic Load Balancer (ELB)?**

- A) To store data across multiple Availability Zones
- B) To distribute incoming traffic across multiple EC2 instances
- C) To provide DNS resolution for domain names
- D) To cache content at edge locations

> **Answer: B** -- An Elastic Load Balancer distributes incoming application traffic across multiple targets (EC2 instances, containers, IP addresses) in one or more Availability Zones. This improves availability and fault tolerance.

---

### Question 5

**Which of the following are benefits of Amazon EC2 Auto Scaling? (Select TWO)**

- A) It automatically encrypts all data on EC2 instances
- B) It adds more instances when demand increases
- C) It removes instances when demand decreases to save costs
- D) It replaces Amazon RDS for database management
- E) It manages IAM policies automatically

> **Answer: B, C** -- Auto Scaling adds instances when traffic rises (scale out) and removes them when traffic drops (scale in). This ensures you have enough capacity during peaks and do not pay for idle resources during low demand. It does not handle encryption, databases, or IAM.

---

### Question 6

**A company has a workload that runs for exactly 5 minutes every hour to process uploaded files. Which compute service is the MOST cost-effective?**

- A) Amazon EC2 On-Demand (running 24/7)
- B) Amazon EC2 Reserved Instance
- C) AWS Lambda
- D) Amazon EC2 Spot Instance

> **Answer: C** -- AWS Lambda charges per request and per millisecond of compute time. For a workload that runs only 5 minutes per hour, Lambda is far cheaper than keeping an EC2 instance running 24/7. You pay for 5 minutes of compute per hour instead of 60 minutes.

---

### Question 7

**What is the difference between Amazon RDS and Amazon DynamoDB?**

- A) RDS is managed; DynamoDB is not managed
- B) RDS is relational (SQL); DynamoDB is non-relational (NoSQL)
- C) DynamoDB supports MySQL and PostgreSQL; RDS does not
- D) They are the same service with different names

> **Answer: B** -- RDS is a managed relational database service that supports SQL engines like MySQL, PostgreSQL, MariaDB, Oracle, and SQL Server. DynamoDB is a managed NoSQL (key-value and document) database. Both are fully managed -- the difference is relational vs. non-relational.

---

### Question 8

**Which component of the AWS Global Infrastructure helps reduce latency by caching content closer to users?**

- A) Availability Zones
- B) Regions
- C) Edge Locations
- D) Local Zones

> **Answer: C** -- Edge locations are data centers used by Amazon CloudFront to cache copies of content near end users. There are more edge locations than Regions or Availability Zones. When a user requests content, it is served from the nearest edge location for low latency.

---

### Question 9

**A company wants to create a private network in AWS with two subnets -- one accessible from the internet and one that is not. What does this describe?**

- A) An Amazon S3 bucket policy
- B) A VPC with a public subnet and a private subnet
- C) An Elastic Load Balancer configuration
- D) An Amazon Route 53 routing policy

> **Answer: B** -- A VPC (Virtual Private Cloud) can contain public subnets (with internet access via an Internet Gateway) and private subnets (no direct internet access). This is a common architecture: web servers go in public subnets, databases go in private subnets.

---

### Question 10

**Which AWS service provides a fully managed message queue that decouples application components?**

- A) Amazon SNS
- B) Amazon SQS
- C) AWS Step Functions
- D) Amazon EventBridge

> **Answer: B** -- Amazon SQS (Simple Queue Service) is a fully managed message queue. It decouples components by letting one component send messages and another process them independently. SNS is a pub/sub notification service. Step Functions orchestrates workflows. EventBridge routes events between services.

## Key Takeaways

- Know what each service does in one sentence -- the exam tests breadth, not deep configuration
- Memorize the categories: Compute (EC2, Lambda, ECS), Storage (S3, EBS, EFS), Database (RDS, DynamoDB), Networking (VPC, Route 53, CloudFront, ELB)
- Lambda is the answer for short, event-driven, serverless workloads
- S3 is the answer for unstructured data storage -- EBS is for EC2 block storage
- RDS = SQL relational databases; DynamoDB = NoSQL key-value databases
