# Practice Questions: Security & Compliance

This domain covers **30% of the CLF-C02 exam** -- the second-heaviest domain. Questions focus on the Shared Responsibility Model, IAM, security services, encryption, and compliance programs. Below are 10 realistic practice questions.

## Tips for Security Questions

- **Shared Responsibility** questions appear constantly -- know which layer belongs to AWS vs. the customer
- **IAM questions** test Users vs. Groups vs. Roles vs. Policies -- know when to use each
- **"Which service does X?"** questions test your ability to match security services to their purpose

---

## Practice Questions

### Question 1

**A security engineer needs to ensure all API calls in an AWS account are logged for audit purposes. Which service should they use?**

- A) Amazon CloudWatch
- B) AWS CloudTrail
- C) Amazon GuardDuty
- D) AWS Config

> **Answer: B** -- AWS CloudTrail records all API calls made in your AWS account. It captures who made the call, what action was taken, when it happened, and from where. CloudWatch monitors metrics and logs. GuardDuty detects threats. Config tracks resource configuration changes.

---

### Question 2

**Under the Shared Responsibility Model, which of the following is AWS responsible for? (Select TWO)**

- A) Configuring security group rules
- B) Physical security of data centers
- C) Managing IAM user passwords
- D) Maintaining the hypervisor that runs EC2 instances
- E) Encrypting customer data at rest

> **Answer: B, D** -- AWS handles physical security and the hypervisor layer. Customers configure security groups, manage IAM passwords, and choose whether to encrypt their data. Remember: if you can configure it in the console, it is your responsibility.

---

### Question 3

**A developer accidentally committed AWS access keys to a public GitHub repository. What is the FIRST thing they should do?**

- A) Delete the GitHub repository
- B) Rotate or deactivate the exposed access keys immediately in IAM
- C) Wait to see if anyone uses them
- D) Change the account password

> **Answer: B** -- The immediate action is to deactivate or delete the exposed access keys in IAM and create new ones. Deleting the GitHub repo does not help because the keys are already in public commit history. Waiting is dangerous -- automated bots scan GitHub for exposed keys within minutes.

---

### Question 4

**Which AWS service protects web applications from common exploits like SQL injection and cross-site scripting?**

- A) AWS Shield
- B) Amazon GuardDuty
- C) AWS WAF (Web Application Firewall)
- D) Amazon Inspector

> **Answer: C** -- AWS WAF lets you create rules that block common attack patterns like SQL injection and cross-site scripting (XSS). Shield protects against DDoS attacks. GuardDuty detects threats at the account level. Inspector scans for software vulnerabilities on EC2 instances.

---

### Question 5

**An administrator wants to require all IAM users to use multi-factor authentication (MFA) before accessing AWS resources. What should they do?**

- A) Enable MFA on the root account only
- B) Create an IAM policy that denies all actions unless MFA is present
- C) Use AWS Shield to enforce MFA
- D) Enable MFA in Amazon S3 bucket settings

> **Answer: B** -- You can create an IAM policy with a condition that denies access unless `aws:MultiFactorAuthPresent` is true. This forces users to authenticate with MFA before performing any actions. Root account MFA protects only the root account, not other IAM users.

---

### Question 6

**What is the difference between encryption at rest and encryption in transit?**

- A) There is no difference -- they are the same thing
- B) At rest protects stored data; in transit protects data moving between systems
- C) At rest protects data moving over the network; in transit protects stored data
- D) At rest uses passwords; in transit uses firewalls

> **Answer: B** -- Encryption at rest protects data stored on disk (S3 objects, EBS volumes, RDS databases). Encryption in transit protects data moving over the network using protocols like TLS/SSL (HTTPS). Both are important and tested on the exam.

---

### Question 7

**Which IAM best practice should you follow for the root user account?**

- A) Use the root user for daily administrative tasks
- B) Share root user credentials with the team
- C) Enable MFA on the root user and use it only for account-level tasks
- D) Delete the root user after creating IAM users

> **Answer: C** -- The root user has unrestricted access to everything. Enable MFA, lock away the credentials, and use it only for tasks that require root (like changing account settings or closing the account). Create IAM users for daily work. You cannot delete the root user.

---

### Question 8

**A company must prove to auditors that AWS meets SOC 2 and ISO 27001 standards. Where can they download these compliance reports?**

- A) AWS Trusted Advisor
- B) AWS Config
- C) AWS Artifact
- D) AWS CloudTrail

> **Answer: C** -- AWS Artifact is a self-service portal where you can download AWS compliance reports, SOC reports, ISO certifications, and PCI attestations. These documents prove that AWS infrastructure meets industry standards.

---

### Question 9

**Which service continuously monitors your AWS environment for malicious activity and unauthorized behavior using machine learning?**

- A) AWS WAF
- B) Amazon GuardDuty
- C) AWS CloudTrail
- D) Amazon Macie

> **Answer: B** -- Amazon GuardDuty uses machine learning, anomaly detection, and threat intelligence to continuously monitor for malicious activity. It analyzes CloudTrail logs, VPC Flow Logs, and DNS logs to detect threats. Macie specifically discovers sensitive data in S3 buckets.

---

### Question 10

**An application running on EC2 needs to access a DynamoDB table. What is the MOST secure way to grant this access?**

- A) Store access keys in the application code
- B) Store access keys in an environment variable on the EC2 instance
- C) Attach an IAM role to the EC2 instance with a policy granting DynamoDB access
- D) Use the root user credentials

> **Answer: C** -- IAM roles provide temporary, automatically rotated credentials to EC2 instances. This is more secure than embedding access keys (which are long-term and can be leaked). Never use root credentials for applications.

## Key Takeaways

- Shared Responsibility Model questions appear across the entire exam -- not just this domain
- Know the purpose of each security service: GuardDuty (threat detection), WAF (web exploits), Shield (DDoS), Inspector (vulnerability scans), Macie (sensitive data), Artifact (compliance reports)
- IAM Roles are always the answer for granting AWS services access to other services -- never embed access keys
- Explicit Deny in IAM always overrides Allow -- this is a favorite exam question
