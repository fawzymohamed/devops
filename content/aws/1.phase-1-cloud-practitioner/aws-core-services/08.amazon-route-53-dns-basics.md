# Amazon Route 53 DNS Basics

**Amazon Route 53** is AWS's **Domain Name System (DNS)** service. It translates human-readable domain names (like `www.example.com`) into IP addresses (like `192.0.2.1`) that computers use to find each other. Route 53 also lets you register domains, route traffic to AWS resources, and check the health of your endpoints.

The name "Route 53" refers to TCP/UDP port 53, which is the standard port for DNS traffic.

## How DNS Works

When you type `www.example.com` in a browser:

1. Your browser asks a **DNS resolver** (usually your ISP or a public resolver like 8.8.8.8) for the IP address
2. The resolver queries **root name servers** to find who manages `.com`
3. The resolver queries the `.com` **TLD (Top-Level Domain) servers** to find who manages `example.com`
4. The resolver queries the **authoritative name server** for `example.com` (this is Route 53 if you use it)
5. Route 53 returns the IP address
6. Your browser connects to that IP address

This entire process typically takes milliseconds. DNS resolvers cache results to speed up repeat lookups.

## Hosted Zones

A **hosted zone** is a container for DNS records for a single domain. When you register a domain or transfer DNS to Route 53, you create a hosted zone.

| Type | Description | Example |
|------|-------------|---------|
| **Public hosted zone** | Routes internet traffic for a domain | `example.com` accessible to everyone |
| **Private hosted zone** | Routes traffic within a VPC only | `internal.example.com` accessible only inside your VPC |

Each hosted zone contains **DNS records** that tell Route 53 how to respond to queries.

## DNS Record Types

| Record Type | Maps To | Example |
|-------------|---------|---------|
| **A** | IPv4 address | `example.com` → `192.0.2.1` |
| **AAAA** | IPv6 address | `example.com` → `2001:db8::1` |
| **CNAME** | Another domain name | `www.example.com` → `example.com` |
| **MX** | Mail server | `example.com` → `mail.example.com` (priority 10) |
| **TXT** | Text string (verification, SPF) | `example.com` → `"v=spf1 include:_spf.google.com"` |
| **NS** | Name servers for the hosted zone | `example.com` → `ns-1234.awsdns-56.org` |
| **Alias** | AWS resource (Route 53 extension) | `example.com` → CloudFront distribution |

### CNAME vs. Alias Records

This is a common point of confusion:

| Feature | CNAME | Alias |
|---------|-------|-------|
| **Points to** | Any domain name | AWS resources (CloudFront, ALB, S3, etc.) |
| **Zone apex** | Cannot be used at apex (`example.com`) | Works at apex |
| **DNS query cost** | Standard charges | Free for AWS resource queries |
| **Non-AWS targets** | Yes | No (AWS resources only) |

**Zone apex** is the bare domain (e.g., `example.com` without any subdomain). DNS rules prohibit CNAME records at the apex, but Route 53 Alias records work around this limitation.

**Best practice:** Use Alias records when pointing to AWS resources. They are free, work at the zone apex, and Route 53 automatically updates them if the AWS resource's IP changes.

## Routing Policies

Route 53 offers several routing policies that control how it responds to DNS queries.

### Simple Routing

Returns one or more IP addresses. If multiple values exist, the client picks one randomly. No health checks involved.

**Use when:** You have a single resource (one web server or one ALB).

### Weighted Routing

Distributes traffic across resources based on **weights** you assign. A resource with weight 70 gets 70% of traffic; weight 30 gets 30%.

**Use when:** You want to test a new version with a small percentage of traffic (canary deployment) or distribute load across regions.

### Latency-Based Routing

Routes users to the AWS Region with the **lowest latency** from their location. Route 53 measures latency between the user and each Region.

**Use when:** You have resources in multiple Regions and want users routed to the fastest one.

### Failover Routing

Directs traffic to a **primary** resource. If the primary fails a health check, Route 53 switches to a **secondary** (backup) resource.

**Use when:** You need active-passive disaster recovery. Primary serves all traffic; secondary only activates if primary is down.

### Geolocation Routing

Routes users based on their **geographic location** (continent, country, or US state). A user in Europe gets European servers; a user in Asia gets Asian servers.

**Use when:** You need to serve localized content, comply with data residency rules, or restrict access by country.

### Geoproximity Routing

Routes users to the nearest resource based on geographic distance. You can add a **bias** to expand or shrink the area that routes to each resource.

**Use when:** You want location-based routing with fine-grained control over traffic distribution.

## Health Checks

Route 53 **health checks** monitor the health of your endpoints. They work with routing policies like Failover and Weighted to route traffic away from unhealthy resources.

Health checks can monitor:

- **An endpoint** -- Route 53 sends HTTP, HTTPS, or TCP requests to your server and checks the response
- **Other health checks** -- combine multiple checks with AND/OR logic (calculated health checks)
- **CloudWatch alarms** -- trigger health check failure based on CloudWatch metric thresholds

When a health check fails, Route 53 stops routing traffic to that endpoint and sends it to healthy alternatives instead.

| Setting | Description | Default |
|---------|-------------|---------|
| **Protocol** | HTTP, HTTPS, or TCP | HTTP |
| **Request interval** | 10 or 30 seconds | 30 seconds |
| **Failure threshold** | Consecutive failures to mark unhealthy | 3 |
| **String matching** | Check if response body contains a specific string | Disabled |

## Key Takeaways

- DNS translates domain names to IP addresses; Route 53 is AWS's managed DNS service
- Hosted zones contain DNS records for a domain; public zones serve the internet, private zones serve VPCs
- Common record types: A (IPv4), AAAA (IPv6), CNAME (domain alias), and MX (mail)
- Alias records are a Route 53 extension that work at the zone apex and point to AWS resources for free
- Routing policies control how Route 53 responds: Simple, Weighted, Latency, Failover, and Geolocation
- Failover routing needs health checks to detect when the primary endpoint is down
- Health checks monitor endpoints and automatically route traffic away from unhealthy resources
