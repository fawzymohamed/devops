# Amazon S3 Storage Classes & Features

**Amazon S3 (Simple Storage Service)** is object storage built for the internet. It stores any amount of data -- images, videos, backups, logs, application assets -- and makes it accessible from anywhere via HTTP. S3 offers 99.999999999% (eleven 9s) durability, meaning your data is extremely safe.

## How S3 Works

S3 organizes data into **buckets** and **objects**:

- **Bucket** -- a container for objects. Each bucket has a globally unique name and belongs to one AWS Region.
- **Object** -- a file stored in a bucket. Each object has a key (file path), the data itself, and metadata. A single object can be up to **5 TB**.
- **Key** -- the unique identifier for an object within a bucket (like a file path: `images/photo.jpg`).

```
my-app-bucket/                 (Bucket)
  ├── images/photo.jpg         (Object with key "images/photo.jpg")
  ├── logs/2024-01-15.log      (Object)
  └── backups/db-snapshot.sql  (Object)
```

S3 is **not** a file system. It is a flat key-value store where the key is the object name and the value is the data. Folder-like paths (e.g., `images/`) are just part of the key string.

## S3 Storage Classes

AWS offers several storage classes that trade access speed for cost. Choose based on how often you access your data.

| Storage Class | Access Pattern | Retrieval Time | Relative Cost |
|---------------|---------------|----------------|---------------|
| **S3 Standard** | Frequent access | Milliseconds | Highest |
| **S3 Intelligent-Tiering** | Changing/unknown patterns | Milliseconds | Auto-optimized |
| **S3 Standard-IA** | Infrequent access (monthly) | Milliseconds | Lower storage, retrieval fee |
| **S3 One Zone-IA** | Infrequent, non-critical | Milliseconds | Lower than Standard-IA |
| **S3 Glacier Instant Retrieval** | Rare access (quarterly) | Milliseconds | Much lower |
| **S3 Glacier Flexible Retrieval** | Archive (1-2 times/year) | Minutes to hours | Very low |
| **S3 Glacier Deep Archive** | Long-term archive | 12-48 hours | Lowest |

### S3 Standard

Default class for frequently accessed data. Stores data across at least three Availability Zones. Use for websites, mobile apps, content distribution, and analytics.

### S3 Intelligent-Tiering

Automatically moves objects between access tiers based on usage. It monitors each object and shifts it to a cheaper tier when not accessed for 30 days, then back when accessed again. Charges a small monthly monitoring fee per object. No retrieval charges.

**Best for:** Data with unknown or changing access patterns.

### S3 Standard-IA (Infrequent Access)

Lower storage cost than Standard but charges a fee each time you retrieve data. Requires a minimum storage duration of 30 days and minimum object size of 128 KB for cost efficiency.

**Best for:** Backups, disaster recovery files, data accessed once a month.

### S3 One Zone-IA

Like Standard-IA but stores data in only **one** Availability Zone instead of three. Costs 20% less than Standard-IA. If that AZ goes down, you lose the data.

**Best for:** Easily re-creatable data like thumbnails or replicated copies.

### S3 Glacier Classes

Three Glacier tiers for archive storage:

- **Glacier Instant Retrieval** -- millisecond access, for data accessed once per quarter
- **Glacier Flexible Retrieval** -- minutes to hours, for annual archives
- **Glacier Deep Archive** -- 12-48 hours, for compliance data kept 7-10 years

## Key S3 Features

### Versioning

**Versioning** keeps every version of an object. When you overwrite a file, S3 saves the old version. When you delete a file, S3 adds a **delete marker** instead of removing it. You can restore any previous version.

Enable versioning on a bucket to protect against accidental overwrites and deletions. Once enabled, versioning can be **suspended** but not fully turned off.

### Lifecycle Policies

**Lifecycle policies** automate moving objects between storage classes or deleting them. You define rules based on object age.

Example policy:
- After **30 days** -- move from S3 Standard to S3 Standard-IA
- After **90 days** -- move to S3 Glacier Flexible Retrieval
- After **365 days** -- delete the object

This saves money without manual intervention.

### Replication

S3 can copy objects to another bucket automatically:

- **Cross-Region Replication (CRR)** -- copies to a bucket in a different Region for disaster recovery or latency reduction
- **Same-Region Replication (SRR)** -- copies within the same Region for compliance or log aggregation

Versioning must be enabled on both source and destination buckets.

### Access Control

S3 offers multiple layers of access control:

- **Bucket policies** -- JSON rules attached to the bucket (e.g., allow public read, restrict by IP)
- **IAM policies** -- control which AWS users and roles can access S3
- **Access Control Lists (ACLs)** -- legacy per-object permissions (AWS recommends bucket policies instead)
- **Block Public Access** -- a bucket-level setting that overrides other policies to prevent accidental public exposure

**Best practice:** Enable Block Public Access on all buckets unless you specifically need public access (like hosting a static website).

### Server-Side Encryption

S3 encrypts all new objects by default using **SSE-S3** (S3-managed keys). You can also choose:

- **SSE-KMS** -- AWS Key Management Service manages the keys (gives you audit trail)
- **SSE-C** -- you provide your own encryption keys

## Key Takeaways

- S3 is object storage that organizes data into buckets and objects, not a file system
- Seven storage classes let you balance access speed against cost
- S3 Intelligent-Tiering automatically optimizes cost for data with unpredictable access patterns
- Versioning protects against accidental overwrites and deletions
- Lifecycle policies automate transitioning objects to cheaper storage classes
- Block Public Access prevents accidental exposure of your data
- S3 encrypts all new objects by default with SSE-S3
