# AWS Core Services - Quick Reference

## Amazon EC2 (Elastic Compute Cloud)

### Instance Families

| Family | Codes | Optimized For | Example Use Cases |
|--------|-------|---------------|-------------------|
| **General Purpose** | T, M | Balanced CPU/memory | Web servers, dev environments |
| **Compute Optimized** | C | High-performance CPU | Batch processing, gaming servers |
| **Memory Optimized** | R, X, z | Large RAM | Redis, in-memory analytics |
| **Storage Optimized** | I, D, H | High sequential I/O | Data warehouses, HDFS |
| **Accelerated Computing** | P, G, Inf | GPU / custom accelerators | ML training, video encoding |

### Instance Naming

```
m5.xlarge
│ │  └── Size (nano → micro → small → medium → large → xlarge → 2xlarge...)
│ └──── Generation
└────── Family
```

### Pricing Models

| Model | Commitment | Savings | Best For |
|-------|-----------|---------|----------|
| **On-Demand** | None | 0% | Short-term, unpredictable |
| **Reserved** | 1-3 years | Up to 72% | Steady-state 24/7 workloads |
| **Spot** | None | Up to 90% | Fault-tolerant, interruptible |
| **Savings Plans** | 1-3 years | Up to 72% | Flexible across instance types |

## Amazon S3 (Simple Storage Service)

### Storage Classes

| Class | Access Pattern | Retrieval | Cost |
|-------|---------------|-----------|------|
| **Standard** | Frequent | Milliseconds | Highest |
| **Intelligent-Tiering** | Unknown/changing | Milliseconds | Auto-optimized |
| **Standard-IA** | Infrequent (monthly) | Milliseconds | Lower + retrieval fee |
| **One Zone-IA** | Infrequent, non-critical | Milliseconds | 20% less than Standard-IA |
| **Glacier Instant** | Rare (quarterly) | Milliseconds | Much lower |
| **Glacier Flexible** | Archive (1-2x/year) | Minutes to hours | Very low |
| **Glacier Deep Archive** | Long-term archive | 12-48 hours | Lowest |

### Key Features

| Feature | What It Does |
|---------|-------------|
| **Versioning** | Keeps all versions; delete adds marker, not removal |
| **Lifecycle Policies** | Auto-transition or delete objects based on age |
| **CRR / SRR** | Cross-Region or Same-Region replication |
| **Block Public Access** | Prevents accidental public exposure |
| **SSE-S3** | Default encryption for all new objects |

## Amazon VPC (Virtual Private Cloud)

### Public vs. Private Subnets

| Aspect | Public Subnet | Private Subnet |
|--------|--------------|----------------|
| **Route to IGW** | Yes | No |
| **Public IPs** | Yes | No |
| **Use for** | Web servers, load balancers | Databases, app servers |
| **Internet access** | Direct (bidirectional) | Outbound only via NAT Gateway |

### Key Components

| Component | Purpose |
|-----------|---------|
| **Internet Gateway (IGW)** | Connects VPC to the internet |
| **NAT Gateway** | Outbound internet for private subnets |
| **Route Table** | Rules that direct traffic by destination |
| **VPC Peering** | Connect two VPCs (non-transitive) |

### Security Groups vs. NACLs

| Feature | Security Group | NACL |
|---------|---------------|------|
| **Level** | Instance | Subnet |
| **Stateful** | Yes | No |
| **Rules** | Allow only | Allow + Deny |
| **Default inbound** | Deny all | Allow all |
| **Evaluation** | All rules together | In order by rule number |

## Amazon RDS (Relational Database Service)

### Supported Engines

MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, **Amazon Aurora**

### Aurora Highlights

- MySQL-compatible (5x throughput) and PostgreSQL-compatible (3x throughput)
- Auto-scales storage from 10 GB to 128 TB
- 6 copies across 3 AZs

### Multi-AZ vs. Read Replicas

| Feature | Multi-AZ | Read Replicas |
|---------|----------|---------------|
| **Purpose** | High availability | Read scaling |
| **Replication** | Synchronous | Asynchronous |
| **Serves traffic** | No (standby only) | Yes (read queries) |
| **Failover** | Automatic | Manual promotion |
| **Cross-Region** | No | Yes |
| **Max replicas** | 1 standby | 5 (15 for Aurora) |

## AWS Lambda (Serverless)

### Key Limits

| Limit | Value |
|-------|-------|
| Max execution time | 15 minutes |
| Max memory | 10 GB |
| Package size (zipped) | 50 MB |
| Concurrent executions | 1,000/Region (adjustable) |

### Pricing

- **Requests:** $0.20 per 1M (first 1M free/month)
- **Duration:** per GB-second (memory x time)
- **Free tier:** 1M requests + 400K GB-seconds/month

### Common Triggers

S3 events, API Gateway, SQS, DynamoDB Streams, CloudWatch Events, SNS, Cognito

### Cold Starts

New execution environment setup adds 100ms to seconds of latency. Reduce with small packages, lighter runtimes (Node.js/Python), or Provisioned Concurrency.

## Elastic Load Balancing & Auto Scaling

### Load Balancer Types

| Type | Layer | Protocol | Best For |
|------|-------|----------|----------|
| **ALB** | 7 (HTTP) | HTTP, HTTPS, gRPC | Web apps, path/host routing |
| **NLB** | 4 (TCP) | TCP, UDP, TLS | Ultra-low latency, static IPs |
| **CLB** | 4 & 7 | HTTP, TCP | Legacy (not recommended) |

### Auto Scaling Capacity

```
Minimum ≤ Desired ≤ Maximum
```

### Scaling Policies

| Policy | How It Works |
|--------|-------------|
| **Target Tracking** | Thermostat: keep metric at target (e.g., 50% CPU) |
| **Step Scaling** | Different amounts based on distance from threshold |
| **Simple Scaling** | Fixed amount on alarm trigger |
| **Scheduled** | Change capacity at specific times |

## Amazon CloudFront (CDN)

### Architecture

```
User → Edge Location (450+) → Regional Edge Cache → Origin (S3, ALB, custom)
```

### TTL (Time to Live)

| Setting | Default |
|---------|---------|
| Default TTL | 24 hours |
| Minimum TTL | 0 seconds |
| Maximum TTL | 365 days |

### Key Concepts

- **Cache hit:** served from edge (fast)
- **Cache miss:** fetched from origin, then cached
- **Invalidation:** force-discard cached content
- **Cache busting:** versioned filenames (preferred over invalidation)
- **OAC (Origin Access Control):** keeps S3 private, serves only via CloudFront

## Amazon Route 53 (DNS)

### Record Types

| Type | Maps To |
|------|---------|
| **A** | IPv4 address |
| **AAAA** | IPv6 address |
| **CNAME** | Another domain (no zone apex) |
| **MX** | Mail server |
| **TXT** | Text (SPF, verification) |
| **Alias** | AWS resource (works at apex, free queries) |

### CNAME vs. Alias

| Feature | CNAME | Alias |
|---------|-------|-------|
| Zone apex | No | Yes |
| Points to | Any domain | AWS resources only |
| Query cost | Standard | Free |

### Routing Policies

| Policy | Use Case |
|--------|----------|
| **Simple** | Single resource, no health checks |
| **Weighted** | Canary deployments, split traffic by % |
| **Latency** | Route to lowest-latency Region |
| **Failover** | Active-passive DR (requires health checks) |
| **Geolocation** | Route by user's country/continent |
| **Geoproximity** | Route by geographic distance with bias |

## Interview Quick Hits

| Question | Answer |
|----------|--------|
| What are the five EC2 instance families? | General Purpose (T/M), Compute Optimized (C), Memory Optimized (R/X), Storage Optimized (I/D), Accelerated Computing (P/G) |
| When would you use Spot Instances? | Fault-tolerant workloads like batch processing; up to 90% savings but AWS can reclaim with 2-min warning |
| How does S3 Intelligent-Tiering work? | Monitors access patterns and auto-moves objects between frequent/infrequent tiers; no retrieval fees |
| What makes a subnet public vs. private? | A public subnet has a route table entry pointing to an Internet Gateway; a private subnet does not |
| Security Groups vs. NACLs? | Security groups are stateful, instance-level, allow-only. NACLs are stateless, subnet-level, allow + deny |
| Multi-AZ vs. Read Replicas in RDS? | Multi-AZ = high availability (sync replication, auto failover, no read traffic). Read Replicas = read scaling (async, serves reads) |
| What is a Lambda cold start? | Latency from creating a new execution environment when a function hasn't been called recently |
| ALB vs. NLB? | ALB is Layer 7 (HTTP path/host routing). NLB is Layer 4 (TCP/UDP, ultra-low latency, static IPs) |

## Quick Decision Guide

**Choose EC2 when:** You need full control over servers, OS, or long-running processes.

**Choose Lambda when:** You run event-driven, short-lived tasks and want zero server management.

**Choose S3 when:** You store files (images, backups, logs) -- not a database.

**Choose RDS when:** You need a managed relational database with SQL.

**Choose ALB when:** You route HTTP traffic by path or host. **Choose NLB when:** You need raw TCP/UDP speed.

**Choose CloudFront when:** You serve content globally and want caching at the edge.

**Choose Route 53 when:** You manage DNS, domain routing, or need health-checked failover.
