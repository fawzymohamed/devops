# AWS Cloud Adoption Framework (CAF)

The **AWS Cloud Adoption Framework (CAF)** is a guide that helps organizations plan, structure, and execute their move to the cloud. It covers both the business and technical sides of cloud adoption so nothing gets overlooked.

Moving to the cloud is not just a technical project. It changes how teams work, how budgets are managed, and how applications are built. CAF gives you a structured way to handle all of these changes.

## The Six Perspectives

CAF organizes cloud adoption guidance into **six perspectives**, split into two groups.

### Business Capabilities

| Perspective | Who It Serves | Focus |
|-------------|---------------|-------|
| **Business** | CEO, CFO, COO | Align cloud investments with business goals. Ensure cloud spending drives real value. |
| **People** | CIO, HR leaders | Manage organizational change. Train teams, define new roles, and build a cloud-ready culture. |
| **Governance** | CIO, CTO, CFO | Control costs, manage risk, and ensure compliance. Set up cloud governance frameworks. |

### Technical Capabilities

| Perspective | Who It Serves | Focus |
|-------------|---------------|-------|
| **Platform** | CTO, architects | Build a scalable cloud platform. Design landing zones, set up networking, and choose services. |
| **Security** | CISO, security teams | Protect data and workloads. Define security controls, encryption, and incident response. |
| **Operations** | IT operations | Run cloud workloads reliably. Set up monitoring, incident management, and automation. |

Think of it this way: the **Business** group answers "Why move?" and "Who is affected?" The **Technical** group answers "How do we build it?" and "How do we keep it safe and running?"

- **Technology** -- Migrate existing workloads, modernize legacy applications, and adopt cloud-native services
- **Process** -- Move from waterfall to agile delivery, adopt CI/CD pipelines, and improve operational practices
- **Organization** -- Restructure teams, build new skills, and create a culture of cloud ownership
- **Product** -- Use data and cloud capabilities to create new products and improve customer experiences

## How Organizations Use CAF

### Step 1: Assess Readiness

Evaluate your current state across all six perspectives. Identify gaps in skills, processes, tools, and governance. AWS provides assessment tools and questionnaires to help.

### Step 2: Build a Roadmap

Based on the gaps, create an action plan. Prioritize what needs to change first -- this might be training your team, setting up governance, or building a landing zone.

### Step 3: Execute and Iterate

Start migrating workloads while continuously improving your cloud practices. Use the CAF perspectives as checkpoints to make sure nothing falls behind.

## Key Takeaways

- AWS CAF is a **planning framework**, not a migration tool -- it helps you organize and prepare for cloud adoption
- Six perspectives cover both business (Business, People, Governance) and technical (Platform, Security, Operations) needs
- Four transformation domains: Technology, Process, Organization, and Product
- Start with a readiness assessment, then build a roadmap before migrating
