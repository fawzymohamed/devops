# AWS Snow Family

The **AWS Snow Family** is a set of physical devices that help you move large amounts of data to AWS and run compute workloads at the edge. When transferring petabytes of data over the internet would take weeks or months, you load it onto a Snow device and ship it to AWS instead.

## Why Physical Data Transfer?

A simple math problem explains it. If you have 100 TB of data and a 1 Gbps network connection, transferring everything over the internet takes **about 12 days** of continuous transfer -- assuming no other traffic uses that connection. At 100 Mbps, it takes **over 100 days**.

Snow Family devices solve this by letting you copy data locally, then physically ship the device to an AWS data center where the data is uploaded to S3.

## The Three Device Types

| Feature | Snowcone | Snowball Edge | Snowmobile |
|---------|----------|---------------|------------|
| **Storage** | 8 TB usable (HDD) or 14 TB (SSD) | Up to 80 TB (Storage Optimized) | Up to 100 PB |
| **Weight** | 4.5 lbs (2.1 kg) | ~50 lbs (22 kg) | 45-foot shipping container |
| **Edge compute** | 2 vCPUs, 4 GB memory | Up to 104 vCPUs, 416 GB memory | No |
| **Best for** | Small, remote locations | Petabyte-scale migrations, edge processing | Exabyte-scale data center moves |

### Snowcone

The **smallest** Snow device. It fits in a backpack and runs on battery power or a standard wall outlet.

- Ideal for **remote and harsh environments** -- military operations, oil rigs, field research
- Supports **AWS DataSync** for online data transfer when connectivity is available
- Can collect and process IoT sensor data at the edge

### Snowball Edge

The **workhorse** of the Snow Family. It comes in two variants:

**Storage Optimized** -- Up to 80 TB of storage. Best for large-scale data migrations, backups, and disaster recovery.

**Compute Optimized** -- More processing power (up to 104 vCPUs). Best for machine learning inference, video processing, and industrial automation at the edge.

Both variants can run **EC2 instances** and **Lambda functions** locally. This means you can process data on the device before shipping it to AWS -- useful when you need to filter, compress, or analyze data at the edge.

### Snowmobile

A **45-foot shipping container** pulled by a semi-trailer truck. It transfers up to **100 petabytes** per trip.

- AWS drives it to your data center, connects it to your network, and you load data onto it
- Used for the largest migrations -- moving entire data centers to AWS
- Includes GPS tracking, 24/7 video surveillance, and an optional security escort

## Edge Computing with Snow

Snow devices are not just for data transfer. They also bring AWS compute to locations without reliable internet:

- **Factories** -- Process manufacturing data locally for quality control
- **Ships and aircraft** -- Run applications where there is no internet connection
- **Disaster zones** -- Deploy temporary compute in areas with damaged infrastructure

## When to Use Snow Family

| Scenario | Recommended Device |
|----------|-------------------|
| Small data collection in remote areas | Snowcone |
| Migrating 10-80 TB of data | Snowball Edge |
| Edge computing with heavy processing | Snowball Edge Compute Optimized |
| Migrating an entire data center (petabyte+) | Multiple Snowball Edge devices or Snowmobile |

## Key Takeaways

- Snow Family devices **physically move data** to AWS when network transfer is too slow
- **Snowcone** is portable (4.5 lbs) and best for small, remote deployments
- **Snowball Edge** handles petabyte-scale migrations and runs EC2/Lambda at the edge
- **Snowmobile** is a shipping container for exabyte-scale data center moves
- All Snow devices use **256-bit encryption** and tamper-evident enclosures for security
