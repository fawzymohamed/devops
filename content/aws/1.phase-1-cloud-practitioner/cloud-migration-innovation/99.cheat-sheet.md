# Cloud Migration & Innovation - Quick Reference

## AWS Cloud Adoption Framework (CAF)

### Six Perspectives

| Group | Perspective | Who It Serves | Focus |
|-------|-------------|---------------|-------|
| **Business** | Business | CEO, CFO, COO | Align cloud investments with business goals |
| **Business** | People | CIO, HR leaders | Organizational change, training, cloud culture |
| **Business** | Governance | CIO, CTO, CFO | Cost control, risk management, compliance |
| **Technical** | Platform | CTO, architects | Cloud architecture, landing zones, services |
| **Technical** | Security | CISO, security teams | Data protection, encryption, incident response |
| **Technical** | Operations | IT operations | Monitoring, incident management, automation |

### Four Transformation Domains

| Domain | What Changes |
|--------|-------------|
| **Technology** | Migrate and modernize workloads |
| **Process** | Waterfall to agile, adopt CI/CD |
| **Organization** | Restructure teams, build cloud skills |
| **Product** | Data-driven innovation, new customer experiences |

**CAF usage:** Assess readiness (identify gaps) --> Build a roadmap --> Execute and iterate.

## Migration Strategies (6 Rs)

| Strategy | Nickname | Effort | Cloud Benefit | Example |
|----------|----------|--------|---------------|---------|
| **Rehost** | Lift and shift | Low | Low | Move on-prem web server to EC2 as-is |
| **Replatform** | Lift, tinker, shift | Low-Med | Medium | Move self-managed MySQL to RDS |
| **Repurchase** | Drop and shop | Medium | Medium | Replace on-prem email with Microsoft 365 |
| **Refactor** | Re-architect | High | High | Break monolith into microservices on ECS + Lambda |
| **Retire** | Shut it down | None | Cost savings | Decommission unused reporting tool |
| **Retain** | Keep as-is | None | None (yet) | Mainframe with multi-year modernization plan |

**Practical approach:** Retire unused apps first --> Rehost the bulk for speed --> Replatform/Refactor high-value apps over time.

## AWS Migration Hub

| Capability | What It Does |
|------------|-------------|
| **Discover** | Find on-prem servers and their configurations |
| **Group** | Organize servers into applications (e.g., web + app + DB = "CRM App") |
| **Plan** | Assess which migration strategy fits each app |
| **Track** | Monitor progress from multiple migration tools in one dashboard |

### Discovery Methods

| Method | How It Works |
|--------|-------------|
| **Agentless** | VMware virtual appliance collects VM config data |
| **Agent-based** | Lightweight agent on each server for deeper metrics and dependency mapping |

### Integrated Migration Tools

| Service | What It Migrates |
|---------|-----------------|
| **AWS Application Migration Service (MGN)** | Servers (lift-and-shift to EC2) |
| **AWS Database Migration Service (DMS)** | Databases (to RDS, Aurora, DynamoDB, Redshift) |

**Two workflows:** Discover first (full inventory, then migrate) or Migrate first (start moving, group as you go).

**Setup:** Choose a **home Region** -- all tracking data is stored there.

## AWS Snow Family

| Feature | Snowcone | Snowball Edge | Snowmobile |
|---------|----------|---------------|------------|
| **Storage** | 8 TB (HDD) / 14 TB (SSD) | Up to 80 TB (Storage Opt.) | Up to 100 PB |
| **Weight** | 4.5 lbs | ~50 lbs | 45-foot shipping container |
| **Edge compute** | 2 vCPUs, 4 GB RAM | Up to 104 vCPUs, 416 GB RAM | No |
| **Best for** | Small remote locations, IoT | Petabyte migrations, edge processing | Exabyte-scale data center moves |

### Snowball Edge Variants

| Variant | Strength | Use Case |
|---------|----------|----------|
| **Storage Optimized** | Up to 80 TB storage | Large data migrations, backups |
| **Compute Optimized** | Up to 104 vCPUs | ML inference, video processing at edge |

Both run **EC2 instances** and **Lambda functions** locally on the device.

### When to Use Snow

| Scenario | Device |
|----------|--------|
| Small data in remote/harsh environments | Snowcone |
| 10-80 TB migration | Snowball Edge |
| Heavy edge compute | Snowball Edge Compute Optimized |
| Entire data center (petabyte+) | Multiple Snowball Edges or Snowmobile |

All Snow devices use **256-bit encryption** and tamper-evident enclosures.

## AWS Outposts & Hybrid Solutions

### Why Outposts?

- **Data residency** -- regulations require data on-premises
- **Low latency** -- factory equipment or trading systems need sub-millisecond response
- **Local processing** -- process data locally, send results to the cloud

### Form Factors

| Type | Size | Scale | Best For |
|------|------|-------|----------|
| **Outposts Racks** | 42U rack | 1-96 racks | Data centers (EC2, ECS, EKS, EBS, S3, RDS, ALB) |
| **Outposts Servers** | 1U or 2U | Individual servers | Retail stores, branch offices, factories (EC2, ECS, IoT Greengrass) |

Outposts connects to a **parent AWS Region** for remote management, updates, and access to additional services. Same AWS APIs, Console, and CLI.

### Other Hybrid Services

| Service | Purpose |
|---------|---------|
| **AWS Outposts** | Run AWS services on-premises with managed hardware |
| **AWS Local Zones** | Extend Regions to metro areas for low latency |
| **AWS Wavelength** | Deploy at telecom 5G edge locations |
| **AWS Snow Family** | Physical data transfer and disconnected edge compute |

## Interview Quick Hits

| Question | Answer |
|----------|--------|
| What are the six CAF perspectives? | Business, People, Governance (business side) and Platform, Security, Operations (technical side) |
| Name the 6 Rs of migration. | Rehost, Replatform, Repurchase, Refactor, Retire, Retain |
| What is the difference between Rehost and Replatform? | Rehost moves apps with zero changes (lift and shift). Replatform makes small optimizations like switching to a managed database |
| What is AWS Migration Hub? | A central dashboard to discover, plan, and track migrations -- it does not perform migrations itself |
| When would you use Snow Family instead of network transfer? | When transferring petabytes of data over the internet would take weeks or months due to limited bandwidth |
| What is the difference between Snowcone and Snowball Edge? | Snowcone is portable (4.5 lbs, 8-14 TB) for small remote deployments. Snowball Edge handles up to 80 TB and runs EC2/Lambda at the edge |
| Why would you choose AWS Outposts? | When data residency rules, low-latency needs, or local processing requirements prevent running workloads entirely in the cloud |
| Which AWS AI service builds chatbots? | Amazon Lex -- it uses the same technology that powers Alexa's natural language understanding |
