# AWS Migration Hub

**AWS Migration Hub** gives you a single dashboard to discover servers, plan migrations, and track progress across multiple AWS migration tools. Instead of switching between different service consoles, you see everything in one place.

## What Migration Hub Does

Migration Hub solves a common problem: when you migrate dozens or hundreds of servers, you need a central view of what has been migrated, what is in progress, and what is left.

| Capability | What It Does |
|------------|-------------|
| **Discover** | Find servers in your data center and understand their configurations |
| **Group** | Organize discovered servers into applications (e.g., "CRM App" = web server + app server + database) |
| **Plan** | Assess which migration strategy fits each application |
| **Track** | Monitor migration progress from multiple AWS tools in one dashboard |

## Discovery Tools

Before migrating, you need to know what you have. Migration Hub connects with discovery tools that scan your on-premises environment.

### AWS Application Discovery Service

This service collects data about your on-premises servers:

- **Agentless discovery** -- Deploys a virtual appliance in your VMware environment to collect VM configuration data (CPU, memory, disk, network)
- **Agent-based discovery** -- Installs a lightweight agent on each server for deeper data like running processes, network connections, and performance metrics

The collected data helps you understand dependencies between servers. For example, if Server A talks to Server B on port 3306, they are probably an application and its MySQL database.

## Migration Workflows

Migration Hub supports two approaches:

### Discover First

1. Run discovery tools to find all servers
2. Group servers into applications
3. Assess each application and pick a migration strategy
4. Start migration and track progress

**Best for:** Organizations that want a complete inventory before moving anything.

### Migrate First

1. Start migrating servers immediately using AWS migration tools
2. Group and organize servers in Migration Hub as migrations proceed
3. Track progress in real time

**Best for:** Organizations that already know their environment and want to move fast.

## Integrated Migration Tools

Migration Hub does not perform migrations itself. It tracks progress from these services:

| Service | What It Migrates |
|---------|-----------------|
| **AWS Application Migration Service (MGN)** | Servers -- lift-and-shift to EC2 instances |
| **AWS Database Migration Service (DMS)** | Databases -- move data to RDS, Aurora, DynamoDB, or Redshift |

Each tool sends status updates to Migration Hub automatically. You see every server and database migration on a single dashboard with status indicators like "not started," "in progress," and "completed."

## Related Services

| Service | Purpose |
|---------|---------|
| **Migration Hub Strategy Recommendations** | Suggests migration strategies (rehost, replatform, refactor) for each application |
| **Migration Hub Orchestrator** | Automates migration workflows with predefined templates |
| **Migration Hub Refactor Spaces** | Helps refactor monolithic applications into microservices incrementally |

## Key Takeaways

- Migration Hub is a **tracking dashboard**, not a migration tool -- it shows progress from other AWS migration services
- Use **Application Discovery Service** to scan your on-premises environment before migrating
- Two workflows: **discover first** (plan carefully) or **migrate first** (move fast)
- AWS Application Migration Service and Database Migration Service report their status to Migration Hub automatically
- Choose a **home Region** during setup -- this is where your tracking data lives
