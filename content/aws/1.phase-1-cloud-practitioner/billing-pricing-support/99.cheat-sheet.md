# Billing, Pricing & Support - Quick Reference

## Pricing Models

| Model | Commitment | Savings | Best For |
|-------|-----------|---------|----------|
| **On-Demand** | None | 0% | Unpredictable workloads, dev/test |
| **Reserved Instances** | 1-3 years | Up to 72% | Steady-state production workloads |
| **Spot Instances** | None | Up to 90% | Fault-tolerant, interruptible jobs |
| **Savings Plans** | 1-3 years ($/hour) | Up to 72% | Flexible across instance types/services |

**Standard RIs** lock to one instance type. **Convertible RIs** allow instance family changes (smaller discount).

**Savings Plan types:** Compute (most flexible -- EC2, Lambda, Fargate) and EC2 Instance (specific family + Region).

### Data Transfer Pricing

| Direction | Cost |
|-----------|------|
| **Data in** (internet to AWS) | Free |
| **Data out** (AWS to internet) | Charged per GB (tiered) |
| **Between services** (same Region) | Often free or low-cost |
| **Between Regions** | Charged per GB |

## AWS Free Tier

| Type | Duration | Examples |
|------|----------|----------|
| **Always Free** | Never expires | Lambda (1M requests/mo), DynamoDB (25 GB), SNS (1M publishes/mo), SQS (1M requests/mo) |
| **12 Months Free** | 12 months from account creation | EC2 (750 hrs t3.micro/mo), S3 (5 GB), RDS (750 hrs db.t3.micro/mo) |
| **Trials** | Varies by service | SageMaker (2 months), GuardDuty (30 days), Inspector (15 days) |

- The 12-month clock starts when you **create your account**, not first use
- Exceeding limits charges **On-Demand rates** -- AWS does NOT auto-stop resources
- Free Tier limits are **per account**, not per Region

## Cost Management Tools

| Tool | When to Use | Key Feature |
|------|-------------|-------------|
| **Pricing Calculator** | **Before** deployment | Estimate future costs (calculator.aws, no account needed) |
| **Cost Explorer** | **After** deployment | Visualize actual spending, filter by service/tag/account, 12-month history |
| **AWS Budgets** | **Ongoing** | Set alerts at cost thresholds, trigger automated actions |
| **Cost and Usage Report (CUR)** | **Detailed analysis** | Line-item billing data exported to S3 for BI tools |
| **Trusted Advisor** | **Ongoing** | Automated best practice checks across 5 categories |

### AWS Budgets - Budget Types

| Type | Tracks |
|------|--------|
| **Cost budget** | Dollar spending (most common) |
| **Usage budget** | Service usage (e.g., EC2 hours, S3 GB) |
| **Reservation budget** | RI and Savings Plan utilization |
| **Savings Plans budget** | Savings Plan coverage % |

**Zero-spend budget:** Alerts at the first dollar of charges -- ideal for Free Tier accounts.

### Cost Explorer Capabilities

- Filter by service, account, Region, tag, instance type
- Forecast up to 12 months ahead
- Right-sizing recommendations for EC2
- Custom reports you can save and share

## AWS Support Plans

| Feature | Basic | Developer | Business | Enterprise |
|---------|-------|-----------|----------|------------|
| **Price** | Free | From $29/mo | From $100/mo | From $15,000/mo |
| **Best for** | Learning | Dev/test | Production | Business-critical |
| **Technical support** | None | Email (business hours) | 24/7 phone, chat, email | 24/7 phone, chat, email |
| **Trusted Advisor** | Core checks (~7) | Core checks (~7) | Full (400+) | Full (400+) |
| **TAM** | No | No | No | Yes (dedicated) |
| **Support API** | No | No | Yes | Yes |
| **Contacts** | N/A | 1 primary | Unlimited | Unlimited |

**Enterprise On-Ramp** ($5,500/mo): Pool of TAMs, 30-min critical response, annual reviews.

### Response Times

| Severity | Developer | Business | Enterprise |
|----------|-----------|----------|------------|
| General guidance | 24 hours | 24 hours | 24 hours |
| System impaired | 12 hours | 12 hours | 12 hours |
| Production impaired | N/A | 4 hours | 4 hours |
| Production down | N/A | 1 hour | 1 hour |
| Business-critical down | N/A | N/A | **15 minutes** |

## Trusted Advisor - Five Categories

| Category | What It Checks |
|----------|---------------|
| **Cost Optimization** | Idle EC2 instances, underutilized EBS, unassociated Elastic IPs, idle load balancers |
| **Performance** | High-utilization EC2, CloudFront optimization, EBS throughput |
| **Security** | MFA on root, S3 public access, overly permissive security groups, unrotated IAM keys |
| **Fault Tolerance** | RDS Multi-AZ, EBS snapshots, single-AZ concentration |
| **Service Limits** | Approaching quotas for EC2, VPC, EBS, IAM |

- Basic/Developer: ~7 core checks (security + some service limits)
- Business/Enterprise: 400+ full checks
- Trusted Advisor **recommends** but does NOT auto-fix
- Dashboard color codes: green (OK), yellow (warning), red (action needed)

## Service Quotas

| Resource | Default Quota | Scope |
|----------|--------------|-------|
| VPCs per Region | 5 | Per Region |
| S3 buckets | 100 | Per account |
| Elastic IPs per Region | 5 | Per Region |
| IAM users | 5,000 | Per account |
| Lambda concurrent executions | 1,000 | Per Region |

- **Adjustable** quotas: request increase via Service Quotas console
- **Non-adjustable** quotas: fixed technical limits (e.g., IAM policy size)
- Hitting a quota **blocks new creation** but does NOT affect existing resources
- Monitor with **CloudWatch alarms** and **Trusted Advisor** service limit checks

## Consolidated Billing & Cost Allocation Tags

### Consolidated Billing (AWS Organizations)

- Management account receives **one bill** for all member accounts
- Combined usage qualifies for **volume discount** tiers (e.g., S3 tiered pricing)
- **Reserved Instances and Savings Plans** shared across accounts
- Free Tier is NOT per account -- one set of limits for the entire organization

### Cost Allocation Tags

| Type | Created By | Example |
|------|-----------|---------|
| **AWS-generated** | AWS (automatic) | `aws:createdBy` |
| **User-defined** | You (manual) | `project=web-app`, `team=marketing`, `environment=production` |

Both types must be **activated** in the Billing console to appear in cost reports.

**Tagging best practices:** Define standard keys early, enforce with AWS Config or SCPs, tag everything, keep naming consistent.

## Interview Quick Hits

| Question | Answer |
|----------|--------|
| What are the four AWS pricing models? | On-Demand (no commitment), Reserved Instances (1-3 year, specific instance), Spot (spare capacity, interruptible), Savings Plans (1-3 year $/hour commitment, flexible) |
| What is the difference between Reserved Instances and Savings Plans? | RIs lock you to a specific instance type and Region. Savings Plans commit a $/hour spend but let you change instance types, Regions, and services |
| What happens when you exceed Free Tier limits? | You pay standard On-Demand rates. AWS does NOT auto-stop your resources |
| What is the difference between Cost Explorer and the Pricing Calculator? | Pricing Calculator estimates costs before deployment. Cost Explorer analyzes actual spending after deployment |
| Which support plan includes a TAM? | Enterprise (dedicated TAM) and Enterprise On-Ramp (pool of TAMs). Developer and Business do not include a TAM |
| What are the five Trusted Advisor categories? | Cost Optimization, Performance, Security, Fault Tolerance, and Service Limits |
| How does consolidated billing save money? | Combines usage across all accounts in an Organization to hit higher volume discount tiers and share RI/Savings Plan benefits |
| What is the difference between adjustable and non-adjustable quotas? | Adjustable quotas can be increased via the Service Quotas console. Non-adjustable quotas are fixed technical limits set by AWS |
