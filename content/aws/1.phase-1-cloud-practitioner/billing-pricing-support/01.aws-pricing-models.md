# AWS Pricing Models

AWS uses a **pay-as-you-go** model by default -- you only pay for the resources you consume, with no upfront contracts. Beyond the default, AWS offers several pricing models that trade flexibility for deeper discounts.

### On-Demand

Pay for compute by the second or hour with **no upfront cost** and **no long-term commitment**. You start and stop instances whenever you want.

- **Discount:** None (full price)
- **Commitment:** None
- **Best for:** Unpredictable workloads, development, testing, short-term projects

### Reserved Instances (RIs)

Commit to using a **specific instance type** in a **specific Region** for 1 or 3 years. In return, you get a significant discount.

- **Discount:** Up to 72% off On-Demand
- **Commitment:** 1 or 3 years
- **Payment options:** All upfront (biggest discount), partial upfront, or no upfront
- **Best for:** Steady-state workloads like production databases and always-on applications

**Standard RIs** lock you into one instance type. **Convertible RIs** let you change instance families during the term for a smaller discount.

### Spot Instances

Use AWS **spare compute capacity** at steep discounts. The catch: AWS can reclaim your instance with a **2-minute warning** when demand rises.

- **Discount:** Up to 90% off On-Demand
- **Commitment:** None
- **Risk:** Instance can be interrupted at any time
- **Best for:** Batch processing, data analysis, CI/CD builds, stateless web servers, any workload that can handle interruption

### Savings Plans

Commit to a **consistent dollar amount per hour** of compute usage for 1 or 3 years. Unlike RIs, you are not locked to a specific instance type.

- **Discount:** Up to 72% off On-Demand
- **Commitment:** 1 or 3 years ($/hour)
- **Types:** Compute Savings Plans (most flexible -- EC2, Lambda, Fargate) and EC2 Instance Savings Plans (specific instance family in a Region)
- **Best for:** Organizations that want RI-level savings with more flexibility

## Other Pricing Concepts

### Volume Discounts

As your usage grows, your **per-unit cost drops**. For example, S3 storage costs less per GB at higher usage tiers. This applies automatically -- no commitment needed.

### Data Transfer Pricing

| Direction | Cost |
|-----------|------|
| **Data in** (internet to AWS) | Free |
| **Data out** (AWS to internet) | Charged per GB (tiered pricing) |
| **Between services** (same Region) | Often free or low-cost |
| **Between Regions** | Charged per GB |

### Per-Service Pricing

Each AWS service has its own pricing dimensions:

| Service | You Pay For |
|---------|------------|
| **EC2** | Instance hours, storage, data transfer |
| **S3** | Storage (per GB), requests, data transfer out |
| **Lambda** | Number of requests + compute duration (per ms) |
| **RDS** | Instance hours, storage, I/O requests |

## Choosing the Right Model

| Workload Type | Best Model | Why |
|---------------|-----------|-----|
| Dev/test environments | On-Demand | No commitment, start/stop freely |
| Production databases | Reserved Instances | Steady usage, maximum savings |
| Batch processing | Spot Instances | Tolerates interruption, huge savings |
| Mixed workloads | Savings Plans | Flexibility across instance types |
| Unknown/new projects | On-Demand | Figure out needs first, commit later |

## Key Takeaways

- **On-Demand** is the default -- pay full price, no commitment, maximum flexibility
- **Reserved Instances** save up to 72% but lock you to a specific instance type and Region
- **Spot Instances** save up to 90% using spare capacity, but AWS can reclaim them anytime
- **Savings Plans** offer RI-level discounts with more flexibility across instance types and services
- Data transfer **into** AWS is free; data transfer **out** is charged
- Start with On-Demand, then move steady workloads to Reserved Instances or Savings Plans
