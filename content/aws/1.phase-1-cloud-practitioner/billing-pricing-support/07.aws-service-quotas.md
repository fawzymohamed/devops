# AWS Service Quotas

Every AWS account has **service quotas** (previously called service limits) -- maximum values for the number of resources you can create. These quotas protect you from accidental over-provisioning and ensure fair access for all customers.

## Why Quotas Exist

- **Prevent runaway costs** -- A misconfigured script could launch thousands of EC2 instances without quotas stopping it
- **Ensure fair sharing** -- AWS infrastructure is shared among millions of customers
- **Protect your account** -- Limits act as a safety net against accidental or unauthorized resource creation

## Common Default Quotas

| Resource | Default Quota | Scope |
|----------|--------------|-------|
| **VPCs per Region** | 5 | Per Region |
| **EC2 On-Demand instances** | Varies by instance type (typically 5-20) | Per Region |
| **S3 buckets per account** | 100 | Per account (global) |
| **Elastic IPs per Region** | 5 | Per Region |
| **IAM users per account** | 5,000 | Per account (global) |
| **Lambda concurrent executions** | 1,000 | Per Region |
| **RDS instances per Region** | 40 | Per Region |

**Important:** Most quotas are **per Region**. You get 5 VPCs in us-east-1 **and** 5 VPCs in eu-west-1 independently.

## Adjustable vs. Non-Adjustable

| Type | Description | Example |
|------|-------------|---------|
| **Adjustable** | You can request an increase | EC2 instance limits, VPC count, S3 buckets |
| **Non-adjustable** | Fixed by AWS, cannot be changed | Max size of an IAM policy (6,144 characters), max name length |

Most quotas are adjustable. Non-adjustable quotas are typically hard technical limits.

## Viewing and Managing Quotas

### Service Quotas Console

The **Service Quotas console** is the central place to:

- **View all quotas** for any AWS service
- **See current usage** compared to the limit
- **Request increases** for adjustable quotas
- **Track request history** and status

### Requesting a Quota Increase

1. Open the **Service Quotas** console
2. Search for the service (e.g., "EC2" or "VPC")
3. Find the specific quota you need to increase
4. Click **Request quota increase**
5. Enter your desired new value
6. Submit the request

Most requests are approved within a few hours to a few days. Some large increases may require a review by AWS.

### Monitoring Quotas with CloudWatch

You can set up **CloudWatch alarms** that alert you when your usage approaches a quota. This gives you time to request an increase before you hit the limit.

**Example:** If your VPC limit is 5 and you have 4, set an alarm at 80% utilization to get notified before you are blocked from creating a new VPC.

Trusted Advisor also includes **service limit checks** that warn you when you are approaching certain quotas.

## What Happens When You Hit a Quota

When you reach a service quota:

- **New resource creation fails** -- you get an error message saying you have reached the limit
- **Existing resources are NOT affected** -- your running instances, databases, and other resources continue to work normally
- **API calls return errors** -- any API or CLI command that tries to create the resource returns a "LimitExceededException" or similar error

**Your account is NOT suspended or penalized.** You simply need to request an increase or clean up unused resources.

## Key Takeaways

- AWS service quotas are **maximum limits** on the number of resources you can create per account or per Region
- Quotas protect against runaway costs and ensure fair resource sharing
- Most quotas are **per Region** -- you get separate limits in each AWS Region
- **Adjustable quotas** can be increased through the Service Quotas console
- Hitting a quota blocks new resource creation but does **not affect existing resources**
- Use **CloudWatch alarms** and **Trusted Advisor** to monitor usage and get warnings before hitting limits
