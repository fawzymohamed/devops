# AWS Trusted Advisor

**AWS Trusted Advisor** is an automated tool that scans your AWS environment and compares it against best practices. It tells you where you can save money, improve performance, close security gaps, increase fault tolerance, and avoid hitting service limits.

Think of it as an automated consultant that continuously reviews your AWS account and flags problems.

### Cost Optimization

Finds resources you are paying for but not fully using.

| Check | What It Finds |
|-------|--------------|
| Idle EC2 instances | Instances with low CPU usage over 14 days |
| Underutilized EBS volumes | Volumes with very low read/write activity |
| Unassociated Elastic IPs | IP addresses allocated but not attached to a running instance (you pay for unused EIPs) |
| Idle load balancers | Load balancers with no active backend targets |
| Reserved Instance optimization | RIs that do not match your actual usage |

### Performance

Identifies settings that slow down your application.

| Check | What It Finds |
|-------|--------------|
| High utilization EC2 | Instances consistently running near 100% CPU |
| CloudFront optimization | CDN settings that could improve delivery speed |
| EBS throughput | Volumes that could benefit from a higher-performance type |

### Security

Flags configurations that leave your account exposed.

| Check | What It Finds |
|-------|--------------|
| MFA on root account | Whether MFA is enabled on your root user |
| S3 bucket permissions | Buckets that allow public access |
| Security group rules | Overly permissive inbound rules (e.g., 0.0.0.0/0 on SSH) |
| IAM access keys | Keys that have not been rotated in 90+ days |
| EBS/RDS encryption | Volumes and databases without encryption at rest |

### Fault Tolerance

Checks if your architecture can survive failures.

| Check | What It Finds |
|-------|--------------|
| RDS Multi-AZ | Databases not configured for automatic failover |
| EBS snapshots | Volumes without recent backups |
| EC2 Availability Zones | Instances concentrated in a single AZ |
| Load balancer health checks | Missing or misconfigured health checks |

### Service Limits

Warns you before you hit AWS **service quotas** (the maximum number of resources you can create).

| Check | What It Finds |
|-------|--------------|
| EC2 instance limit | Approaching the maximum number of instances per Region |
| VPC limit | Near the maximum number of VPCs |
| EBS volume limit | Close to the maximum number of volumes |
| IAM policy limit | Approaching policy size or count limits |

## Core Checks vs. Full Checks

| Plan Level | Checks Available |
|------------|-----------------|
| **Basic & Developer** | ~7 core checks (key security checks + some service limits) |
| **Business & Enterprise** | 400+ full checks across all five categories |

Core checks include essentials like MFA on root account, S3 bucket permissions, and a few service limit checks. The full suite adds all cost optimization, performance, and fault tolerance checks.

## How to Use Trusted Advisor

1. **Open Trusted Advisor** in the AWS Management Console
2. **Review the dashboard** -- checks are color-coded: green (no issues), yellow (warning), red (action recommended)
3. **Drill into red and yellow items** to see affected resources and recommendations
4. **Take action** -- Trusted Advisor tells you what to fix, but you must fix it yourself
5. **Refresh checks** periodically -- some checks update automatically, others require a manual refresh

## Key Takeaways

- Trusted Advisor scans your AWS environment against best practices across **five categories**: Cost Optimization, Performance, Security, Fault Tolerance, and Service Limits
- **Basic and Developer** plans get ~7 core checks (mostly security and service limits)
- **Business and Enterprise** plans unlock **400+ checks** with detailed recommendations
- Trusted Advisor **recommends** actions but does NOT automatically fix issues
- Common recommendations include terminating idle instances, enabling MFA, and fixing overly permissive security groups
- Use the color-coded dashboard (green/yellow/red) to prioritize which issues to fix first
