# Consolidated Billing & Cost Allocation Tags

When organizations use multiple AWS accounts, they need a way to combine billing and track costs by project or team. **Consolidated billing** merges all account charges into one bill. **Cost allocation tags** let you label resources so you can see exactly who is spending what.

## Consolidated Billing

Consolidated billing is a feature of **AWS Organizations**. The **management account** (formerly called master account) receives a single bill that covers charges from all member accounts.

Each member account operates independently, but all charges roll up to the management account's bill.

### Benefits

**Single bill** -- One invoice, one payment method. Finance teams manage one relationship with AWS instead of dozens.

**Volume discounts** -- AWS services like S3 use tiered pricing. More usage means a lower per-unit cost.

| Account | S3 Usage | Billed Separately | Billed Together |
|---------|----------|------------------|-----------------|
| Account A | 50 TB | Higher rate (first 50 TB tier) | Combined rate |
| Account B | 60 TB | Higher rate (first 50 TB tier) | Combined rate |
| Account C | 40 TB | Higher rate (first 40 TB tier) | Combined rate |
| **Total** | **150 TB** | Each pays first-tier rates | All benefit from 150 TB tier |

At 150 TB combined, the per-GB price drops to a lower tier than any single account could reach alone.

**Reserved Instance and Savings Plan sharing** -- If Account A buys a Reserved Instance but does not fully use it, the unused capacity can apply to matching usage in Account B or C. This maximizes your RI investment across the organization.

### What Is NOT Shared

- **Free Tier** -- The organization gets one set of Free Tier limits, not one per account
- **Support plans** -- Each account can have a different support plan level
- **Account isolation** -- Member accounts cannot access each other's resources unless explicitly configured with cross-account IAM roles

## Cost Allocation Tags

Tags are **key-value pairs** that you attach to AWS resources. When activated as **cost allocation tags**, they appear in your billing reports so you can filter and group costs.

### Tag Examples

| Key | Value | Purpose |
|-----|-------|---------|
| `project` | `web-app` | Track spending per project |
| `team` | `marketing` | Track spending per team |
| `environment` | `production` | Separate production vs. development costs |
| `cost-center` | `CC-4200` | Map to internal accounting codes |

### Two Types of Tags

| Type | How Created | Example |
|------|-------------|---------|
| **AWS-generated** | Automatically applied by AWS when you create resources | `aws:createdBy` (who created the resource) |
| **User-defined** | You create and apply them manually to resources | `project=web-app`, `team=marketing` |

Both types must be **activated** in the AWS Billing console before they appear in cost reports.

### Setting Up Cost Allocation Tags

1. **Tag your resources** -- Add key-value pairs to EC2 instances, S3 buckets, RDS databases, and other resources
2. **Activate tags in Billing** -- Go to the Billing console and activate the tags you want to use for cost tracking
3. **View tagged costs** -- Use Cost Explorer to filter and group spending by tag

**Example report view:**

| Tag (project) | EC2 | S3 | RDS | Total |
|---------------|-----|-----|-----|-------|
| web-app | $400 | $50 | $200 | $650 |
| mobile-api | $300 | $30 | $150 | $480 |
| data-pipeline | $100 | $200 | $0 | $300 |

This tells you exactly how much each project costs -- information that is impossible to get without tags.

### Tagging Best Practices

- **Define a tagging strategy early** -- agree on standard keys (project, team, environment) before teams start creating resources
- **Enforce tags** -- use AWS Config rules or SCPs to require tags on new resources
- **Keep it simple** -- start with 3-5 essential tags and add more as needed
- **Use consistent naming** -- decide on lowercase vs. uppercase, hyphens vs. underscores, and stick to it
- **Tag everything** -- untagged resources show up as "uncategorized" in reports, making cost tracking incomplete

## Key Takeaways

- **Consolidated billing** combines charges from all accounts in an AWS Organization into a single bill paid by the management account
- Combined usage unlocks **volume discounts** -- the more your organization uses, the lower the per-unit price
- **Reserved Instances and Savings Plans** can be shared across accounts in the organization
- **Cost allocation tags** are key-value pairs that label resources for cost tracking
- **AWS-generated tags** are applied automatically; **user-defined tags** are created by you -- both must be activated in the Billing console
- Define a **tagging strategy** early and enforce it to get accurate cost breakdowns by project, team, or environment
