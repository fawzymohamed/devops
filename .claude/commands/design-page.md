Design a new page for the LMS: $ARGUMENTS

## Process

1. **Analyze Requirements**
   - What is the purpose of this page?
   - What data does it display?
   - What actions can users take?

2. **Review Existing Patterns**
   - Check existing pages for consistency
   - Review the ui-ux-design skill for patterns
   - Check nuxt-ui skill for component options

3. **Create Wireframe**
   - Describe the layout structure
   - Identify component needs
   - Plan responsive behavior

4. **Implement with Nuxt UI**
   - Use UCard, UButton, UBadge, etc.
   - Follow dark mode color palette
   - Add loading and error states
   - Ensure accessibility

5. **Add Interactivity**
   - Hover states and transitions
   - Mobile menu handling
   - Loading indicators

## Design Checklist

- [ ] Follows existing page patterns
- [ ] Uses Nuxt UI components consistently
- [ ] Has loading skeleton state
- [ ] Has error state handling
- [ ] Has empty state (if applicable)
- [ ] Responsive for mobile/tablet/desktop
- [ ] Accessible (focus states, aria labels)
- [ ] Dark mode colors only
- [ ] `cursor-pointer` on clickable elements

## Common Pages

- **Phase overview**: Grid of topics with progress
- **Topic overview**: List of lessons with completion status
- **Lesson page**: Content with quiz and navigation
- **Progress page**: Stats and phase breakdown
- **Certificate page**: Completion certificate with download

## Output

Provide:
1. Page layout wireframe (ASCII)
2. Component breakdown
3. The Vue page file
4. Any new components needed
